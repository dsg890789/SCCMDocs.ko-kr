# <a name="create-configuration-items-for-windows-devices-with-on-premises-mdm-in-configuration-manager"></a>Configuration Manager에서 온-프레미스 MDM을 사용 하 여 Windows 장치에 대 한 구성 항목 만들기

*적용 대상: Configuration Manager (현재 분기)*

Configuration Manager **Windows 8.1 및 windows 10** 구성 항목을 사용 하 여 온-프레미스 MDM (모바일 장치 관리)을 사용 하 여 관리 하는 windows 장치에 대 한 설정을 관리 합니다.

Configuration Manager의 준수 설정에 대 한 일반적인 내용은 다음 문서를 참조 하세요.

- [준수 설정 시작](/configmgr/compliance/get-started/get-started-with-compliance-settings)

- [준수 설정 계획 및 구성](/configmgr/compliance/plan-design/plan-for-and-configure-compliance-settings)

## <a name="create-a-configuration-item"></a>구성 항목 만들기

1. Configuration Manager 콘솔에서 **자산 및 호환성** 작업 영역으로 이동 하 고 **호환성 설정**을 확장 한 다음 **구성 항목** 노드를 선택 합니다.

1. 리본의 **홈** 탭에 있는 **만들기** 그룹에서 **구성 항목 만들기**를 선택합니다.

1. 에 **일반** 의 페이지는 **구성 항목 만들기 마법사**, 다음 정보를 지정 합니다.

    - **이름**:이 구성 항목을 식별 하는 고유 이름입니다.

    - **설명**: 사용에 대 한 추가 정보를 제공 하는 선택적 설명입니다.

    - **Configuration Manager 클라이언트를 사용 *하지 않고* 관리 되는 장치에 대 한 설정**에서 **Windows 8.1 및 Windows 10**을 선택 합니다.

    - **범주**: Configuration Manager 콘솔에서 구성 항목을 검색 하 고 필터링 하는 데 도움이 되는 범주를 만들고 할당할 수 있습니다.

1. 마법사의 **지원 되는 플랫폼** 페이지에서이 구성 항목을 평가할 특정 Windows 플랫폼을 선택 합니다.

1. **장치 설정** 페이지에서 구성 하려는 설정 그룹을 선택 합니다. 사용 가능한 설정에 대 한 자세한 내용은 [설정 참조](#bkmk_setref)를 참조 하세요.

    > [!TIP]
    > 원하는 설정이 나열 되지 않은 경우 **기본 설정 그룹에 없는 추가 설정을 구성**하는 옵션을 선택 합니다. 이 옵션은 마법사에 **추가 설정** 페이지를 추가 합니다.

1. 각 설정 페이지에서 필요한 설정을 구성 합니다. 설정에서 지원 되는 경우 **비규격 설정을 수정할**수도 있습니다. 장치가이 구성 항목을 준수 하지 않는 것으로 평가 하는 경우 설정을 준수 하는 것으로 재구성 합니다.

    각 설정 그룹에 대해 설정이 비규격 인 경우 보고 하는 심각도를 구성할 수도 있습니다. **보고서의 비호환 심각도** 옵션에 대해 다음 값 중 하나를 선택 합니다.

    - **없음**: 이 준수 규칙에 실패한 디바이스가 Configuration Manager 보고서에 오류 심각도를 보고하지 않습니다.

    - **정보**

    - **경고**

    - **위험**

    - **위험(이벤트 포함)** : 이 준수 규칙에 실패한 디바이스가 Configuration Manager 보고서에 **위험** 오류 심각도를 보고합니다. 또한 비규격 상태를 응용 프로그램 이벤트 로그에 Windows 이벤트로 기록 합니다.

1. 마법사의 **플랫폼 적용 가능성** 페이지에서 선택한 지원 되는 플랫폼과 호환 되지 않는 설정을 검토 합니다. 뒤로 돌아가서 이러한 설정을 제거 하거나 지원 플랫폼을 변경 하거나 계속 합니다.

    > [!IMPORTANT]
    > 장치는 지원 되지 않는 설정의 준수를 평가 하지 않습니다.

1. 마법사 완료

**자산 및 준수** 작업 영역의 **구성 항목** 노드에서 새 구성 항목을 볼 수 있습니다.

## <a name="bkmk_setref"></a>설정 참조  

다음 섹션에서는 각 그룹에서 사용할 수 있는 특정 설정을 자세히 설명 합니다. Configuration Manager 클라이언트 *없이* 관리 되는 **Windows 8.1 및 Windows 10** 장치에 대 한 **구성 항목 만들기 마법사** 의 **장치 설정** 페이지에서 이러한 설정을 구성 합니다.

### <a name="password"></a>암호

이러한 설정은 Windows 10 이상을 실행 하는 장치 에서만 사용할 수 있습니다.

- **장치에 암호 설정 필요**: 지원 되는 장치에 암호 필요
- **최소 암호 길이 (문자)** : 암호의 최소 길이입니다.
- **암호 만료 일**수: 사용자가 암호를 변경 해야 하기 전 까지의 기간 (일)입니다.
- **저장 한 암호 수**: 이전에 사용한 암호의 재사용을 방지 합니다.
- 다음 **로그온 실패 횟수 후 장치 초기화**:이 수의 로그인 시도가 실패 하면 MDM에서 장치를 초기화 합니다.
- 다음 **유휴 시간 후 장치 잠김**: 장치가 잠기기 전까지 유휴 상태를 유지할 수 있는 시간을 지정 합니다. 사용자 입력이 없는 경우 장치가 유휴 상태입니다.
- **암호 복잡성**: `1234`와 같은 숫자 PIN을 지정할 수 있는지 여부 또는 강력한 암호를 제공 해야 하는지 여부를 선택 합니다.
  - **암호에 필요한 복합 문자 집합 수**: 암호 복잡성이 **강력한**경우 암호에 필요한 문자 유형 수 (대문자, 소문자, 숫자 또는 기호)를 선택 합니다. 이 값은 기본적으로 `2`입니다.
- **Exchange Server에 암호 복구 PIN 전송**

### <a name="device"></a>디바이스

- **화면 캡처**: 사용자가 장치 디스플레이의 스크린샷을 사용할 수 있도록 하려면이 설정을 사용 하도록 설정 합니다. (Windows 10만 해당)
- **진단 데이터 전송**: 분석을 위해 Windows 진단 데이터를 허용 하려면이 설정을 사용 하도록 설정 합니다. (Windows 8.1에만 해당)
- **진단 데이터 제출 허용 (windows 10)** : 사용 안 함, 기본, 고급 또는 전체를 분석에 대 한 windows 진단 데이터의 수준을 설정 합니다. (Windows 10만 해당)
- **지리적**위치: 장치에서 위치 서비스 정보를 사용할 수 있도록 하려면이 설정을 사용 하도록 설정 합니다. (Windows 10만 해당)
- **복사 및 붙여넣기**: 복사 및 붙여넣기를 사용 하 여 앱 간에 데이터를 전송 하려면이 설정을 사용 하도록 설정 합니다. (Windows 10만 해당)
- **공장 재설정**: 사용자가 장치를 초기 설정으로 다시 설정할 수 있도록 하려면이 설정을 사용 하도록 설정 합니다. (Windows 10만 해당)
- **Bluetooth**: 장치의 Bluetooth 기능 사용을 허용 하거나 금지 합니다.
- **Bluetooth 검색 가능 모드**: 다른 bluetooth 장치에서 장치를 검색 하도록 허용 하거나 금지 합니다. (Windows 10만 해당)
- **Bluetooth 광고**: bluetooth 광고 사용을 허용 하거나 금지 합니다. (Windows 10만 해당)
- **음성 녹음**: 장치의 음성 녹음 기능 사용을 허용 하거나 금지 합니다. (Windows 10만 해당)
- **Cortana**: cortana 음성 도우미의 사용을 허용 하거나 금지 합니다. (Windows 10만 해당)
- 알림 **센터 알림**: Windows 10에서 알림 창을 사용 하거나 사용 하지 않도록 설정 합니다. (Windows 10만 해당)
- **지역 설정 수정 (데스크톱만 해당)** : 사용자가 장치에서 국가별 설정을 변경 하는 것을 허용 하거나 금지 합니다.
- **전원 및 절전 모드 설정 수정 (데스크톱만 해당)** : 사용자가 장치에서 전원 및 절전 모드 설정을 변경 하는 것을 허용 하거나 금지 합니다.
- **언어 설정 수정 (데스크톱만 해당)** : 사용자가 장치에서 언어 설정을 변경 하는 것을 허용 하거나 금지 합니다.
- **시스템 시간 수정**: 사용자가 장치 날짜 및 시간을 변경 하는 것을 허용 하거나 금지 합니다.
- **장치 이름 수정**: 사용자가 장치 이름을 변경 하지 못하도록 허용 하거나 허용 하지 않습니다.

### <a name="email-management"></a>메일 관리

다음 설정은 Windows 8.1 및 Windows 10을 실행하는 디바이스에 적용됩니다.  

- **Pop 및 imap 전자 메일**: POP 및 imap 표준을 사용 하는 전자 메일 계정에 대 한 연결을 허용 하거나 금지 합니다.
- **전자 메일을 보관할 최대 시간**: 서버에서 전자 메일을 삭제 하기 전에 유지 하는 기간입니다.
- **허용 된 메시지 형식**: 전자 메일의 **일반 텍스트** 또는 **HTML 및 일반 텍스트**를 지정 합니다.
- **일반 텍스트 전자 메일의 최대 크기 (자동 다운로드**된 경우): 자동으로 다운로드 될 때 일반 텍스트 전자 메일의 최대 크기를 설정 합니다.
- **Html 전자 메일의 최대 크기 (자동 다운로드**된 경우): 자동으로 다운로드 될 때 html 전자 메일의 최대 크기를 설정 합니다.
- **첨부 파일의 최대 크기 (자동 다운로드**된 경우): 자동으로 다운로드 되는 전자 메일 첨부 파일의 최대 크기를 설정 합니다.
- **일정 동기화**: 장치에 대 한 일정 동기화를 허용 하거나 금지 합니다.
- **사용자 지정 전자 메일 계정**: 장치에서 조직 이외의 메일 계정 사용을 허용 하거나 금지 합니다.
- **Windows 메일 앱에서 Microsoft 계정을 옵션**으로 설정: windows mail에서 Microsoft 계정 필요 하지 않도록 하려면이 옵션을 사용 하도록 설정 합니다.

### <a name="store"></a>스토어

이러한 설정은 Windows 10 이상을 실행 하는 장치 에서만 사용할 수 있습니다.

- **응용 프로그램 저장소**: 장치에서 Microsoft Store에 대 한 액세스를 허용 하거나 금지 합니다.
- **응용 프로그램 스토어에 액세스 하려면 암호 입력**: 사용자가 Microsoft Store 앱에 액세스 하기 위해 암호를 입력 하도록 요구 하려면이 옵션을 사용 하도록 설정 합니다.
- **앱에서**바로 구매: 사용자가 앱에서 바로 구매할 수 있도록 허용 하거나 허용 하지 않습니다.
- 스토어에서 시작 된 **앱 시작**: 장치에 미리 설치 되어 있거나 Microsoft Store에서 설치 된 모든 앱을 사용 하지 않도록 설정 합니다.
- **스토어에서 앱 자동 업데이트**: Microsoft Store에서 설치 된 앱을 자동으로 업데이트 하도록 허용 하거나 금지 합니다.
- **시스템 드라이브에 앱 설치**: 일반적으로 `C:` 드라이브인 시스템 드라이브에 앱을 설치 하는 장치를 허용 하거나 금지 합니다.
- **시스템 볼륨에 앱 데이터 설치**: 앱이 시스템 드라이브에 데이터를 저장할 수 있도록 하려면이 옵션을 사용 하도록 설정 합니다.
- **개인 저장소만 사용**: 사용자가 개인 저장소에서 앱을 다운로드 하도록 요구 합니다.
- **게임 DVR**: Windows 게임 녹화 및 브로드캐스팅 사용 안 함

### <a name="browser"></a>브라우저

다음 설정은 Windows 8.1 및 Windows 10을 실행하는 디바이스에 적용됩니다.

- **웹 브라우저 허용**: 장치에서 웹 브라우저 사용을 허용 하거나 금지 합니다.
- 자동 **채우기**: 브라우저에서 자동 완성 설정 변경을 허용 하거나 금지 합니다.
- **액티브 스크립팅**: 브라우저에서 ActiveX 스크립트와 같은 스크립트를 실행할 수 있는지 여부를 허용 하거나 금지 합니다.
- **플러그**인: Internet Explorer에 플러그 인을 추가 하는 것을 허용 하거나 금지 합니다.
- **팝업 차단**: 브라우저 팝업 차단을 허용 하거나 금지 합니다.
- **쿠키**: 장치에 쿠키를 저장 하도록 허용 하거나 금지 합니다.
- **사기 행위 경고**: 잠재적 사기성 웹 사이트의 경고를 허용 하거나 금지 합니다.

### <a name="internet-explorer"></a>Internet Explorer

다음 설정은 Windows 8.1 및 Windows 10을 실행하는 디바이스에 적용됩니다.

- **항상 Do Not Track 헤더 보내기**: 타사 사이트에 검색 정보를 보내는 것을 허용 하거나 금지 합니다.
- **인트라넷 보안 영역**: 인트라넷 보안 영역에 대 한 보안 수준의 할당을 허용 하거나 금지 합니다.
- **인터넷 영역에 대 한 보안 수준**: 높음, 중간, 높음 또는 중간의 인터넷 영역에 대 한 보안 수준을 설정 합니다.
- **인트라넷 영역에 대 한 보안 수준**: 높음, 보통 높음, 중간, 보통 낮음 또는 낮음의 인트라넷 영역에 대 한 보안 수준을 설정 합니다.
- **신뢰할 수 있는 사이트 영역에 대 한 보안 수준**: 높음, 보통 높음, 중간, 보통 낮음 또는 낮음의 신뢰할 수 있는 사이트 영역에 대 한 보안 수준을 설정 합니다.
- **제한 된 사이트 영역의 보안 수준**: 제한 된 사이트 영역에 대 한 보안 수준 (높음)을 설정 합니다.
- **인트라넷 영역에 대 한 네임 스페이스**: 인트라넷 영역에서 추가 하거나 제거할 웹 사이트를 구성 합니다.
- 한 **단어 입력으로 인트라넷 사이트로 이동**: 사용자가 이전 프로토콜 없이 유효한 사이트 이름을 입력 하는 경우 (예: `https://`) Internet Explorer가 자동으로 인트라넷 사이트로 이동 하도록 허용 하거나 금지 합니다.

#### <a name="configured-wireless-network-connections"></a>구성 된 무선 네트워크 연결

1. **모바일 장치 무선 통신 설정 구성** 페이지에서 **추가**를 선택 합니다.

1. **무선 네트워크 연결** 창에서 모바일 장치에 프로 비전 할 무선 연결에 대 한 다음 정보를 지정 합니다.

    - **네트워크 이름 (SSID)** : wi-fi 네트워크의 이름을 입력 합니다.
    - **네트워크 연결**: **인터넷** 또는 **작업**을 선택 합니다.
    - **인증**: 무선 연결에 대 한 인증 방법을 선택 합니다.
        - **열기**
        - **공유**
        - **WPA**
        - **WPA-PSK**
        - **문과**
        - **WPA2-PSK**
    - **데이터 암호화**:이 연결에 사용 되는 암호화 방법을 선택 합니다. 사용 가능한 값은 선택한 **인증** 방법에 따라 달라 집니다.
        - **사용 안 함**
        - **4**
        - **TKIP**
        - **AES**
    - **키 인덱스**: **데이터 암호화** 를 **WEP**로 설정 하는 경우 **1** 에서 **4**사이의 키 인덱스를 선택 합니다.
    - **이 네트워크를 인터넷에 연결**: 무선 네트워크의 모바일 장치를 인터넷에 연결할 수 있도록 프록시 설정을 제공 합니다.
        - **프록시 서버 설정**: **HTTP**, **WAP**및 **소켓** 프록시에 대 한 **서버** 및 **포트** 설정을 구성 합니다.
    - **802.1 x 설정**:
        - **802.1 x 네트워크 액세스 사용**: EAP 유형을 지정 하 여 연결을 보호 합니다.
        - **EAP 유형**: 다음 인증 프로토콜 중 하나를 선택 합니다.
            - **PEAP**
            - **스마트 카드 또는 인증서**

- **주소 표시줄에서 검색 제안 허용**: 검색 구를 입력할 때 검색 엔진에서 사이트를 제안할 수 있습니다.

- **SmartScreen 사용**: 다운로드 한 파일에 악성 코드가 포함 되지 않았는지 확인 합니다.
- **팝업 허용**: 브라우저 팝업을 구성 합니다.
- **쿠키 허용**: 쿠키 사용을 구성 합니다.
- 자동 **채우기 허용**: 브라우저에서 이름, 주소, 전화 등의 저장 된 정보를 사용 하 여 양식을 자동으로 채울 수 있도록 합니다.
- **암호 관리자 허용**: 브라우저에서 웹 사이트에 대 한 암호를 저장 하 고 관리할 수 있도록 합니다.
- **개발자 도구**: Edge의 개발자 도구 기능을 허용 하거나 금지 합니다.
- **확장**: Edge 확장을 허용 하거나 금지 합니다.
- **Inprivate 브라우징**: 기록 또는 쿠키를 저장 하지 않는 inprivate 브라우징을 허용 하거나 금지 합니다.
- **WebRTC LOCALHOST ip 주소**: 사용자가 웹 RTC 프로토콜을 사용 하 여 전화를 걸 때 장치의 localhost ip 주소가 표시 되도록 허용 하거나 금지 합니다.
- **About: flags에 대 한 액세스 차단**: 사용자가 개발자 및 실험적 설정을 포함 하는 `about:flags` 페이지에 액세스 하도록 허용 하거나 금지 합니다.
- **파일에 대 한 smartscreen 프롬프트 재정의**: 잠재적으로 악성 파일 다운로드에 대 한 smartscreen 필터 경고를 무시 하도록 허용 하거나 금지 합니다.
- **Smartscreen 프롬프트 재정의**: 사용자가 잠재적으로 악성 웹 사이트에 대 한 SmartScreen 필터 경고를 무시 하도록 허용 하거나 금지 합니다.
- **첫 실행 URL**: 사용자가 Edge를 처음 열 때 표시할 웹 사이트를 지정 합니다.

### <a name="windows-defender-antivirus"></a>Windows Defender 바이러스 백신

이러한 설정은 Windows 10 이상을 실행 하는 장치 에서만 사용할 수 있습니다.

- **실시간 모니터링 허용**: 맬웨어, 스파이웨어 및 기타 사용자 동의 없이 설치 된 소프트웨어에 대 한 실시간 검사를 사용 하도록 설정 합니다.
- **동작 모니터링 허용**: Defender는 장치에서 의심 스러운 활동의 알려진 특정 패턴을 검사 합니다.
- **네트워크 검사 시스템 사용**: NIS (네트워크 검사 시스템)는 네트워크 기반 익스플로잇 으로부터 장치를 보호 하는 데 도움이 됩니다. Microsoft Endpoint Protection Center에서 알려진 취약성의 서명을 사용하여 악성 트래픽을 검색하고 차단합니다.
- **모든 다운로드 검색**: Defender는 인터넷에서 다운로드 한 모든 파일을 검색 합니다.
- **스크립트 검색 허용**: Defender는 Internet Explorer에서 사용 되는 스크립트를 검색 합니다.
- **파일 및 프로그램 활동 모니터링**: Defender는 장치에서 파일 및 프로그램 활동을 모니터링 합니다.
  - **모니터링**되는 파일: 모니터링할 파일 유형 (모두, 수신 또는 발신)을 선택 합니다.
- **해결 된 맬웨어 추적 일**수: Defender는 지정 된 일 수 동안 해결 된 맬웨어를 계속 추적 합니다. 그런 다음 이전에 영향을 받은 장치를 수동으로 확인할 수 있습니다. 일 수를 0으로 설정 하면 맬웨어는 격리 폴더에 유지 되며 자동으로 제거 되지 않습니다. 최대값은 90입니다.
- **클라이언트 UI 액세스 허용**: 사용자의 Defender 사용자 인터페이스를 숨길지 여부를 제어 합니다. 이 설정을 변경 하면 다음에 장치를 다시 시작할 때 적용 됩니다.
- **시스템 검색 예약**: 선택한 날짜와 시간에 정기적으로 발생 하는 전체 또는 빠른 검색을 선택 합니다.
  - **예약 날짜**: Defender에서 검사를 실행할 때 예약 된 날짜를 선택 합니다.
  - **예약 시간**: Defender에서 검색을 실행 하는 예약 된 시간을 선택 합니다.
- **매일 빠른 검색 예약**: Defender가 매일 빠른 검색을 실행할 예정 시간을 선택 합니다.
- **검색 중 CPU 사용량 제한**: 검색을 실행할 때 Defender에서 사용할 수 있는 프로세서의 백분율을 설정 합니다. 1에서 100 사이의 값을 입력 합니다.
- **보관 파일 검색**: Defender는 .zip 또는 .cab 파일과 같은 압축 된 보관 파일을 검색 합니다.
- **전자 메일 메시지 검색**: Defender는 장치에 도착 했을 때 전자 메일 메시지를 검색 합니다.
- **이동식 드라이브 검색**: DEFENDER는 USB 스틱 같은 이동식 드라이브를 검색 합니다.
- **매핑된 드라이브 검색**: Defender는 네트워크 공유에 매핑된 드라이브를 검색 합니다. 예를 들어 `H:`은 사용자의 개인 드라이브에 매핑됩니다. 드라이브의 파일이 읽기 전용인 경우 Defender는 해당 파일에서 발견 된 맬웨어를 제거할 수 없습니다.
- **네트워크 공유 폴더에서 열린 파일 검색**: 사용자가 공유 네트워크 경로에서 파일을 열면 Defender에서 파일을 검색 합니다. 예: `\\server\share\file.doc` 공유의 파일이 읽기 전용인 경우 Defender는 찾은 맬웨어를 제거할 수 없습니다.
- **서명 업데이트 간격**: Defender가 새 서명 파일을 확인 하는 시간 간격을 선택 합니다.
- **클라우드 보호 허용**: Defender는 Microsoft 클라우드를 사용 하 여 맬웨어 활동에 대 한 정보를 받고, 처음에는 블록 처럼 기능을 사용 하도록 설정 합니다.
- **사용자에 게 샘플 제출 확인**: 파일에 추가 분석이 필요할 수 있는 경우 Defender에 대 한 동작을 선택 합니다. 예를 들어 Defender는 자동으로 파일을 Microsoft에 보내 악성 인지 확인할 수 있습니다.
- 사용자 **동의 없이 설치 된 응용 프로그램 검색**: 잠재적으로 사용자 동의 없이 Defender에 의해 분류 된 소프트웨어를 실행 하지 않도록 장치를 보호 합니다. 이러한 응용 프로그램이 실행 되는 것을 방지 하거나 감사 모드를 사용 하 여 사용자 동의 없이 설치 된 응용 프로그램을 설치 하는 경우를 보고할 수 있습니다.
- **파일 및 폴더 제외**: 제외 목록에 하나 이상의 파일 및 폴더를 추가 합니다. 예를 들어 `C:\Path` 또는 `%ProgramFiles%\Path\filename.exe`로 이름을 지정할 수 있습니다. Defender는 실시간 또는 예약 된 검색에 이러한 파일 및 폴더를 포함 하지 않습니다.
- **파일 확장명 제외**: 제외 목록에 하나 이상의 파일 확장명을 추가 합니다. 예를 들어 `java` 또는 `exe`로 이름을 지정할 수 있습니다. Defender는 실시간 또는 예약 된 검색에 이러한 확장을 포함 하는 파일을 포함 하지 않습니다.
- **예외 처리**: 제외 목록에 특정 프로세스를 추가 합니다. 예: `C:\path\myproc.exe` 이 제외 형식은 `exe`, `com`또는 `scr`확장만 지원 합니다. Defender는 실시간 또는 예약 된 검색에 이러한 프로세스를 포함 하지 않습니다.

### <a name="additional-settings"></a>추가 설정

마법사의 **장치 설정** 페이지에서 **기본 설정 그룹에 없는 추가 설정을 구성**하는 옵션을 선택 하는 경우이 **추가 설정** 페이지를 사용 하 여 해당 설정을 구성할 수 있습니다. **추가** 를 선택한 다음 사용 가능한 모바일 설정 목록에서 선택 합니다. **선택** 을 선택 하 여 기본 제공 설정을 수정 하거나 **설정 만들기** 를 선택 하 여 사용자 지정 레지스트리 값 또는 OMA URI 설정 유형을 만듭니다.
