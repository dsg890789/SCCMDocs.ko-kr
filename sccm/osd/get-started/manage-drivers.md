# <a name="manage-drivers-in-configuration-manager"></a>Configuration Manager에서 드라이버 관리

*적용 대상: System Center Configuration Manager(현재 분기)*

Configuration Manager는 현재 Configuration Manager 환경에서 Windows 디바이스 드라이버를 관리하는 데 사용할 수 있는 드라이버 카탈로그를 제공합니다. 드라이버 카탈로그를 사용하여 Configuration Manager로 디바이스 드라이버를 가져오고, 드라이버를 패키지로 그룹화하고, 이 패키지를 배포 지점에 배포할 수 있습니다. 대상 컴퓨터에 전체 OS를 설치하는 경우 및 부팅 이미지에서 Windows PE를 사용하는 경우 디바이스 드라이버를 사용할 수 있습니다. Windows 디바이스 드라이버는 INF(설치 정보) 파일과 디바이스를 지원하는 데 필요한 추가 파일로 구성됩니다. OS를 배포하는 경우 Configuration Manager가 해당 INF 파일에서 디바이스에 대한 하드웨어 및 플랫폼 정보를 가져옵니다. 



## <a name="BKMK_DriverCategories"></a> 드라이버 범주

디바이스 드라이버를 가져올 때 각 디바이스 드라이버를 범주에 할당할 수 있습니다. 디바이스 드라이버 범주를 사용하면 서로 비슷하게 사용되는 디바이스 드라이버를 드라이버 카탈로그에 함께 그룹화할 수 있습니다. 예를 들어 모든 네트워크 어댑터 디바이스 드라이버를 특정 범주로 설정합니다. 그런 다음, [드라이버 자동 적용](/sccm/osd/understand/task-sequence-steps#BKMK_AutoApplyDrivers) 단계를 포함하는 작업 순서를 만들 때 디바이스 드라이버의 특정 범주를 지정합니다. Configuration Manager에서 하드웨어를 검사하여 Windows 설치 프로그램이 사용할 수 있도록 시스템에서 준비할 해당 드라이버를 이 범주에서 선택합니다.  



## <a name="BKMK_ManagingDriverPackages"></a> 드라이버 패키지

OS 배포를 간소화하기 위해 패키지에서 비슷한 디바이스 드라이버를 그룹화합니다. 예를 들어 네트워크에 있는 각 컴퓨터의 제조업체에 대한 드라이버 패키지를 만듭니다. **드라이버 패키지** 노드에서 직접 드라이버를 드라이버 카탈로그로 가져올 때 드라이버 패키지를 만들 수 있습니다. 드라이버 패키지를 만든 후에 배포 지점에 배포합니다. 그러면 구성 관리자 클라이언트 컴퓨터가 필요에 따라 드라이버를 설치할 수 있습니다. 

다음 사항을 고려하세요.  

- 드라이버 패키지를 만들 경우 패키지의 원본 위치는 다른 드라이버 패키지에서 사용하지 않는 비어 있는 네트워크 공유를 가리켜야 합니다. SMS 공급자는 해당 위치에 대한 **모든 권한**이 있어야 합니다.  

- 드라이버 패키지에 디바이스 드라이버를 추가하면 Configuration Manager가 해당 디바이스 드라이버를 패키지 원본 위치에 복사합니다. 드라이버 카탈로그로 가져와 사용하도록 설정한 디바이스 드라이버만 드라이버 패키지에 추가할 수 있습니다.  

- 기존 드라이버 패키지에서 디바이스 드라이버의 하위 집합을 복사할 수 있습니다. 먼저 새 드라이버 패키지를 만드세요. 그런 다음, 디바이스 드라이버의 하위 집합을 새 패키지에 추가하고 새 패키지를 배포 지점에 배포하세요.  

- 작업 순서를 사용하여 드라이버를 설치하는 경우 500개 미만의 디바이스 드라이버가 포함된 드라이버 패키지를 만듭니다.


### <a name="BKMK_CreatingDriverPackages"></a> 드라이버 패키지 만들기  

> [!IMPORTANT]  
> 드라이버 패키지를 만들려면 다른 드라이버 패키지에서 사용하지 않는 비어 있는 네트워크 폴더가 있어야 합니다. 대부분의 경우 이 절차를 시작하려면 먼저 새 폴더를 만듭니다.  

1. Configuration Manager 콘솔에서 **소프트웨어 라이브러리** 작업 영역으로 이동합니다. **운영 체제**를 확장하고 **드라이버 패키지** 노드를 선택합니다.  

2. 리본의 **홈** 탭에 있는 **만들기** 그룹에서 **드라이버 패키지 만들기**를 선택합니다.  

3. 드라이버 패키지를 설명하는 **이름**을 지정합니다.  

4. 드라이버 패키지에 대한 **설명**을 입력합니다(선택 사항). 이 설명을 통해 드라이버 패키지의 내용이나 용도에 대한 정보를 제공합니다.  

5. **경로** 상자에 드라이버 패키지의 비어 있는 원본 폴더를 지정합니다. 각 드라이버 패키지에는 고유한 폴더를 사용해야 합니다. 이 경로는 네트워크 위치로 필수입니다.  

   > [!IMPORTANT]  
   > 사이트 서버 계정에는 지정된 원본 폴더에 대한 **전체 권한**이 있어야 합니다.  

새 드라이버 패키지에는 드라이버가 포함되어 있지 않습니다. 다음 단계에서는 패키지에 드라이버를 추가합니다.  

**드라이버 패키지** 노드에 여러 패키지가 포함되어 있는 경우 노드에 폴더를 추가하여 패키지를 논리적 그룹으로 구분할 수 있습니다.  


### <a name="BKMK_PackageActions"></a> 드라이버 패키지에 대한 추가 작업  

**드라이버 패키지** 노드에서 드라이버 패키지를 하나 이상 선택하면 드라이버 패키지 관리를 위한 추가 작업을 수행할 수 있습니다. 


#### <a name="create-prestage-content-file"></a>사전 준비된 콘텐츠 파일 만들기
콘텐츠 및 관련 메타데이터를 수동으로 가져오는 데 사용할 수 있는 파일을 만듭니다. 사전 준비된 콘텐츠는 사이트 서버와 드라이버 패키지가 저장되는 배포 지점 사이의 네트워크 대역폭이 낮은 경우에 사용합니다.

#### <a name="delete"></a>삭제
**드라이버 패키지** 노드에서 드라이버 패키지를 제거합니다.

#### <a name="distribute-content"></a>콘텐츠 배포
배포 지점, 배포 지점 그룹 및 컬렉션에 연결된 배포 지점 그룹에 드라이버 패키지를 배포합니다.

#### <a name="manage-access-accounts"></a>액세스 계정 관리
드라이버 패키지에 대한 액세스 계정을 추가, 수정 또는 제거합니다.

패키지 액세스 계정에 대한 자세한 내용은 [Configuration Manager에서 사용되는 계정](/sccm/core/plan-design/hierarchy/accounts)을 참조하세요.

#### <a name="move"></a>이동
드라이버 패키지를 **드라이버 패키지** 노드의 다른 폴더로 이동합니다.

#### <a name="update-distribution-points"></a>배포 지점 업데이트
패키지가 저장된 모든 배포 지점에서 디바이스 드라이버 패키지를 업데이트합니다. 이 작업은 패키지가 마지막으로 배포된 이후 변경된 콘텐츠만 복사합니다.

#### <a name="properties"></a>속성
**속성** 대화 상자를 엽니다. 드라이버의 콘텐츠 및 속성을 검토하고 변경합니다. 예를 들어 드라이버의 이름 및 설명을 변경하고, 드라이버를 사용하도록 설정하거나 사용하지 않도록 설정하고, 드라이버가 실행될 수 있는 플랫폼을 지정합니다.

버전 1810부터 드라이버 패키지에는 **제조업체** 및 **모델**에 대한 메타데이터 필드가 포함됩니다. 일반적인 하우스키핑을 지원하거나, 삭제해도 되는 오래되거나 중복되는 드라이버를 식별하기 위해 이 필드를 사용하여 드라이버 패키지를 정보에 태그합니다. **일반** 탭에서 드롭다운 목록의 기존 값을 선택하거나 새 항목을 만들기 위한 문자열을 입력합니다.

**드라이버 패키지** 노드에서 이러한 필드는 **드라이버 제조업체** 및 **드라이버 모델** 열로 목록에 표시됩니다. 이러한 필드를 검색 조건으로도 사용할 수 있습니다.

## <a name="BKMK_DeviceDrivers"></a> 디바이스 드라이버

드라이버를 배포되는 OS 이미지에 포함하지 않고 대상 컴퓨터에 설치할 수 있습니다. Configuration Manager에서는 Configuration Manager로 가져오는 모든 드라이버에 대한 참조를 포함하는 드라이버 카탈로그를 제공합니다. 드라이버 카탈로그는 **소프트웨어 라이브러리** 작업 영역에 있으며 **드라이버** 와 **드라이버 패키지**의 두 노드로 구성되어 있습니다. **드라이버** 노드에는 드라이버 카탈로그로 가져온 모든 드라이버가 나열됩니다.  


### <a name="BKMK_ImportDrivers"></a> 드라이버 카탈로그에 디바이스 드라이버 가져오기  

OS를 배포할 때 드라이버를 사용하려면 먼저 드라이버 카탈로그로 가져옵니다. 더 효율적으로 관리하려면 OS 배포의 일부로 설치할 드라이버만 가져오세요. 카탈로그에 여러 버전의 드라이버를 저장해 두면 네트워크에서 하드웨어 디바이스 요구 사항이 변경될 경우 기존 드라이버를 조금 더 쉽게 업그레이드할 수 있습니다.  

Configuration Manager는 디바이스 드라이버 가져오기 프로세스의 일환으로 드라이버에 대한 다음 속성을 읽습니다.
- Provider
- 인스턴스
- Version
- 서명
- 지원되는 하드웨어
- 지원되는 플랫폼 정보

기본적으로 드라이버는 드라이버가 지원하는 첫 번째 하드웨어 디바이스를 따라 이름이 지정됩니다. 나중에 디바이스 드라이버의 이름을 바꿀 수 있습니다. 지원되는 플랫폼 목록은 드라이버의 INF 파일에 포함된 정보에 따릅니다. 이 정보의 정확성은 달라질 수 있으므로 드라이버를 카탈로그에 가져온 후 드라이버가 지원되는지 수동으로 확인하세요.  

디바이스 드라이버를 카탈로그에 가져온 후 드라이버 패키지 또는 부팅 이미지 패키지에 디바이스 드라이버를 추가합니다.  

> [!IMPORTANT]  
> 디바이스 드라이버를 **드라이버** 노드의 하위 폴더로 직접 가져올 수는 없습니다. 디바이스 드라이버를 하위 폴더로 가져오려면 먼저 디바이스 드라이버를 **드라이버** 노드로 가져온 후에 하위 폴더로 이동해야 합니다.  


#### <a name="process-to-import-windows-device-drivers-into-the-driver-catalog"></a>Windows 디바이스 드라이버를 드라이버 카탈로그로 가져오는 프로세스  

1. Configuration Manager 콘솔에서 **소프트웨어 라이브러리** 작업 영역으로 이동합니다. **운영 체제**를 확장하고 **드라이버** 노드를 선택합니다.  

2. 리본의 **홈** 탭에 있는 **만들기** 그룹에서 **드라이버 가져오기**를 선택하여 **새 드라이버 가져오기 마법사**를 시작합니다.  

3. **드라이버 찾기** 페이지에서 다음 옵션을 지정합니다.  

    - **다음 네트워크 경로 (UNC)의 모든 드라이버 가져오기**: 특정 폴더의 모든 장치 드라이버를 가져오려면 해당 네트워크 경로를 지정 합니다. 예: `\\servername\share\folder`.  

        > [!NOTE]  
        > 하위 폴더 및 드라이버 INF 파일이 많이 있는 경우 이 프로세스에 시간이 걸릴 수 있습니다.  

    - **특정 드라이버 가져오기**: 폴더에서 특정 드라이버를 가져오려면 Windows 디바이스 드라이버 INF 파일에 대한 네트워크 경로를 지정합니다.  

    - **중복 드라이버에 대한 옵션 지정**: 중복 디바이스 드라이브를 가져올 때 Configuration Manager에서 드라이버 범주를 관리하는 방식을 선택합니다  
        - **드라이버 가져오기 및 새 범주를 기존 범주에 추가**  
        - **드라이버 가져오기 및 기존 범주 유지**  
        - **드라이버 가져오기 및 기존 범주 덮어쓰기**  
        - **드라이버를 가져오지 않음**  

    > [!IMPORTANT]  
    > 드라이버를 가져올 때 사이트 서버에는 폴더에 대한 **읽기** 권한이 있어야 하며, 그렇지 않으면 가져올 수 없습니다.  

4. **드라이버 정보** 페이지에서 다음 옵션을 지정합니다.  

    - **(부팅 이미지용) 스토리지 또는 네트워크 클래스에 없는 드라이버 숨기기**: 이 설정을 사용하여 스토리지 및 네트워크 드라이버만 표시합니다. 이 옵션을 사용하면 부팅 이미지에 일반적으로 필요하지 않은 비디오 드라이버 또는 모뎀 드라이버 같은 다른 드라이버가 숨겨집니다.  

    - **디지털 서명 되지 않은 드라이버 숨기기**: 디지털 서명 된 드라이버만 사용 하는 것이 좋습니다.  

    - 드라이버 목록에서 드라이버 카탈로그로 가져올 드라이버를 선택합니다.  

    - **이들 드라이버를 사용하고 컴퓨터에 설치할 수 있도록 허용**: 컴퓨터에 디바이스 드라이버를 설치할 수 있도록 하려면 이 설정을 선택합니다. 기본적으로 이 옵션은 사용하도록 설정됩니다.  

        > [!IMPORTANT]  
        > 디바이스 드라이버가 문제를 일으키거나 디바이스 드라이버 설치를 일시 중단하려면 드라이버를 가져오는 중에 사용하지 않도록 설정하세요. 드라이버를 가져온 후 사용하지 않도록 설정할 수도 있습니다.  

    - 필터링 목적으로 “데스크톱” 또는 “노트북” 같은 관리 범주에 디바이스 드라이버를 할당하려면 **범주**를 선택합니다. 그런 다음, 기존 범주를 선택하거나 새 범주를 만듭니다. 범주를 사용하여 [드라이버 자동 적용](/sccm/osd/understand/task-sequence-steps#BKMK_AutoApplyDrivers) 작업 순서 단계에서 적용하는 디바이스 드라이버를 제어합니다.  

5. **패키지에 드라이버 추가** 페이지에서 패키지에 드라이버를 추가할지 여부를 선택합니다.  

    - 디바이스 드라이버를 배포하는 데 사용되는 드라이버 패키지를 선택합니다.  

        필요한 경우 **새 패키지**를 선택하여 새 드라이버 패키지를 만듭니다. 새 드라이버 패키지를 만들 때는 다른 드라이버 패키지에서 사용하지 않는 네트워크 공유를 지정합니다.  

    - 패키지가 배포 지점에 이미 배포된 경우 대화 상자에서 **예**를 선택하여 배포 지점에서 부팅 이미지를 업데이트합니다. 디바이스 드라이버가 배포 지점에 배포될 때까지 디바이스 드라이버를 사용할 수 없습니다. **아니요**를 선택하면 부팅 이미지를 사용하기 전에 **배포 지점 업데이트** 작업을 실행합니다. 드라이버 패키지를 배포하지 않은 경우에는 **드라이버 패키지** 노드에서 **콘텐츠 배포** 작업을 사용해야 합니다.  

6. **부팅 이미지에 드라이버 추가** 페이지에서 기존 부팅 이미지에 디바이스 드라이버를 추가할지 여부를 선택합니다.  

    > [!NOTE]  
    > 부팅 이미지에는 스토리지 및 네트워크 드라이버만 추가합니다.  

    - 배포 지점에서 부팅 이미지를 업데이트하려면 대화 상자에서 **예**를 선택합니다. 디바이스 드라이버가 배포 지점에 배포될 때까지 디바이스 드라이버를 사용할 수 없습니다. **아니요**를 선택하면 부팅 이미지를 사용하기 전에 **배포 지점 업데이트** 작업을 실행합니다. 드라이버 패키지를 배포하지 않은 경우에는 **드라이버 패키지** 노드에서 **콘텐츠 배포** 작업을 사용해야 합니다.  

    - 드라이버 하나 이상의 아키텍처와 선택한 부팅 이미지의 아키텍처가 일치하지 않는 경우 Configuration Manager에서 경고가 표시됩니다. 아키텍처가 일치하지 않는 경우 **확인**을 선택합니다. **드라이버 정보** 페이지로 돌아가서 선택한 부팅 이미지의 아키텍처와 일치하지 않는 드라이버의 선택을 취소합니다. 예를 들어 x64 및 x86 부팅 이미지를 선택하는 경우 모든 드라이버가 두 아키텍처를 모두 지원해야 합니다. x64 부팅 이미지를 선택하는 경우에는 모든 드라이버가 x64 아키텍처를 지원해야 합니다.  

        > [!NOTE]  
        > - 아키텍처는 제조업체 INF에서 보고되는 아키텍처를 기준으로 합니다.  
        > - 두 아키텍처를 모두 지원하는 것으로 보고하는 드라이버는 두 부팅 이미지 중 하나로 가져올 수 있습니다.  

    - 네트워크 또는 스토리지 드라이버가 아닌 디바이스 드라이버를 부팅 이미지에 추가하면 Configuration Manager에서 경고가 표시됩니다. 대부분의 경우 이러한 드라이버는 부팅 이미지에 필요하지 않습니다. 부팅 이미지에 해당 드라이버를 추가하려면 **예**를 선택하고, 뒤로 돌아가서 드라이버 선택을 수정하려면 **아니요**를 선택합니다.  

    - 하나 이상의 선택한 드라이버가 적절하게 디지털 서명되어 있지 않으면 Configuration Manager에서 경고가 표시됩니다. 계속하려면 **예**를 선택하고, 뒤로 돌아가서 드라이버 선택을 변경하려면 **아니요**를 선택합니다.  

7. 마법사를 완료합니다.  


### <a name="BKMK_ModifyDriverPackage"></a> 드라이버 패키지에서 디바이스 드라이버 관리  

드라이버 패키지와 부팅 이미지를 수정하려면 다음 절차를 따르십시오. 드라이버를 추가 또는 제거하려면 먼저 **드라이버** 노드에서 해당 드라이버를 찾습니다. 그런 다음, 선택한 드라이버와 연결된 패키지 또는 부팅 이미지를 편집합니다.  

1. Configuration Manager 콘솔에서 **소프트웨어 라이브러리** 작업 영역으로 이동합니다. **운영 체제**를 확장하고 **드라이버** 노드를 선택합니다.  

2. 드라이버 패키지에 추가할 디바이스 드라이버를 선택합니다.  

3. 리본의 **홈** 탭에 있는 **드라이버** 그룹에서 **편집**, **드라이버 패키지**를 차례로 선택합니다.  

4. 디바이스 드라이버를 추가하려면 디바이스 드라이버를 추가할 드라이버 패키지의 확인란을 선택합니다. 디바이스 드라이버를 제거하려면 디바이스 드라이버를 제거할 드라이버 패키지의 확인란을 선택 취소합니다.  

    드라이버 패키지에 연결된 디바이스 드라이버를 추가하는 경우 선택적으로 새 패키지를 만들 수 있습니다. **새 패키지**를 선택하면 **새 드라이버 패키지** 대화 상자가 열립니다.  

5. 패키지가 배포 지점에 이미 배포된 경우 대화 상자에서 **예**를 선택하여 배포 지점에서 부팅 이미지를 업데이트합니다. 디바이스 드라이버가 배포 지점에 배포될 때까지 디바이스 드라이버를 사용할 수 없습니다. **아니요**를 선택하면 부팅 이미지를 사용하기 전에 **배포 지점 업데이트** 작업을 실행합니다. 드라이버 패키지를 배포하지 않은 경우에는 **드라이버 패키지** 노드에서 **콘텐츠 배포** 작업을 사용해야 합니다. 배포 지점에서 드라이버 패키지를 업데이트해야 드라이버를 사용할 수 있습니다.  

    마치면 **확인**을 선택합니다.  


### <a name="BKMK_ManageDriversBootImage"></a> 부팅 이미지에서 디바이스 드라이버 관리  

카탈로그로 가져온 Windows 디바이스 드라이버를 부팅 이미지에 추가할 수 있습니다. 부팅 이미지에 디바이스 드라이버를 추가할 경우 다음 지침을 참조하십시오.  

- 부팅 이미지에는 스토리지 및 네트워크 드라이버만 추가합니다. 일반적으로 다른 종류의 드라이버는 Windows PE에 필요하지 않습니다. 필요하지 않은 드라이버는 부팅 이미지의 크기만 불필요하게 늘릴 뿐입니다.  

- Windows 10용 디바이스 드라이버만 부팅 이미지에 추가합니다. Windows PE의 필수 버전은 Windows 10을 기반으로 합니다.  

- 부팅 이미지의 아키텍처에 적합한 디바이스 드라이버를 사용해야 합니다. 예를 들어 x64 부팅 이미지에 x86 디바이스 드라이버를 추가해서는 안 됩니다.  

#### <a name="process-to-modify-the-device-drivers-associated-with-a-boot-image"></a>부팅 이미지와 연결된 디바이스 드라이버를 수정하는 프로세스  

1. Configuration Manager 콘솔에서 **소프트웨어 라이브러리** 작업 영역으로 이동합니다. **운영 체제**를 확장하고 **드라이버** 노드를 선택합니다.  

2. 드라이버 패키지에 추가할 디바이스 드라이버를 선택합니다.  

3. 리본의 **홈** 탭에 있는 **드라이버** 그룹에서 **편집**, **부팅 이미지**를 차례로 선택합니다.  

4. 디바이스 드라이버를 추가하려면 디바이스 드라이버를 추가할 부팅 이미지의 확인란을 선택합니다. 디바이스 드라이버를 제거하려면 디바이스 드라이버를 제거할 부팅 이미지의 확인란을 선택 취소합니다.  

5. 부팅 이미지가 저장되는 배포 지점을 업데이트하지 않으려는 경우 **마치면 배포 지점 업데이트** 확인란의 선택을 취소합니다. 기본적으로 배포 지점은 부팅 이미지가 업데이트될 때 업데이트됩니다.  

    - 배포 지점에서 부팅 이미지를 업데이트하려면 대화 상자에서 **예**를 선택합니다. 디바이스 드라이버가 배포 지점에 배포될 때까지 디바이스 드라이버를 사용할 수 없습니다. **아니요**를 선택하면 부팅 이미지를 사용하기 전에 **배포 지점 업데이트** 작업을 실행합니다. 드라이버 패키지를 배포하지 않은 경우에는 **드라이버 패키지** 노드에서 **콘텐츠 배포** 작업을 사용해야 합니다.  

    - 드라이버 하나 이상의 아키텍처와 선택한 부팅 이미지의 아키텍처가 일치하지 않는 경우 Configuration Manager에서 경고가 표시됩니다. 아키텍처가 일치하지 않는 경우 **확인**을 선택합니다. **드라이버 정보** 페이지로 돌아가서 선택한 부팅 이미지의 아키텍처와 일치하지 않는 드라이버의 선택을 취소합니다. 예를 들어 x64 및 x86 부팅 이미지를 선택하는 경우 모든 드라이버가 두 아키텍처를 모두 지원해야 합니다. x64 부팅 이미지를 선택하는 경우에는 모든 드라이버가 x64 아키텍처를 지원해야 합니다.  

        > [!NOTE]
        > - 아키텍처는 제조업체 INF에서 보고되는 아키텍처를 기준으로 합니다.  
        > - 두 아키텍처를 모두 지원하는 것으로 보고하는 드라이버는 두 부팅 이미지 중 하나로 가져올 수 있습니다.  

    - 네트워크 또는 스토리지 드라이버가 아닌 디바이스 드라이버를 부팅 이미지에 추가하면 Configuration Manager에서 경고가 표시됩니다. 대부분의 경우 이러한 드라이버는 부팅 이미지에 필요하지 않습니다. 부팅 이미지에 해당 드라이버를 추가하려면 **예**를 선택하고, 뒤로 돌아가서 드라이버 선택을 수정하려면 **아니요**를 선택합니다.  

    - 하나 이상의 선택한 드라이버가 적절하게 디지털 서명되어 있지 않으면 Configuration Manager에서 경고가 표시됩니다. 계속하려면 **예**를 선택하고, 뒤로 돌아가서 드라이버 선택을 변경하려면 **아니요**를 선택합니다.  


### <a name="BKMK_DriverActions"></a> 디바이스 드라이버에 대한 추가 작업  

**드라이버** 노드에서 드라이버를 선택할 때 드라이버 관리를 위한 추가 작업을 수행할 수 있습니다.  

#### <a name="categorize"></a>범주
선택한 드라이버에 대한 관리 범주를 지우거나, 관리하거나, 설정합니다.

#### <a name="delete"></a>삭제
**드라이버** 노드에서 드라이버를 제거하고 연결된 배포 지점에서도 해당 드라이버를 제거합니다.

#### <a name="disable"></a>사용 안 함
드라이버 설치를 허용하지 않습니다. 이 작업은 드라이버를 일시적으로 사용하지 않도록 설정합니다. OS가 배포될 때 사용할 수 없는 드라이버를 작업 순서에서 설치할 수 없습니다. 

> [!Note]  
> 이 작업은 **드라이버 자동 적용** 작업 순서 단계를 사용하여 드라이버를 설치하는 것만 방지합니다.

#### <a name="enable"></a>사용 하도록 설정
OS가 배포될 때 구성 관리자 클라이언트 컴퓨터 및 작업 순서에서 디바이스 드라이버를 설치하도록 허용합니다.

#### <a name="move"></a>이동
디바이스 드라이버를 **드라이버** 노드의 다른 폴더로 이동합니다.

#### <a name="properties"></a>속성
**속성** 대화 상자를 엽니다. 드라이버의 속성을 검토하고 변경합니다. 예를 들어, 이름 및 설명을 변경하고, 사용하도록 설정하거나 사용하지 않도록 설정하고, 실행될 수 있는 플랫폼을 지정합니다.



## <a name="BKMK_TSDrivers"></a> 작업 순서를 사용하여 드라이버 설치  

작업 순서를 사용하여 OS가 배포되는 방식을 자동화합니다. 작업 순서의 각 단계에서는 드라이버 설치와 같은 특정 작업을 수행할 수 있습니다. 다음 두 작업 순서 단계를 사용하여 OS를 배포할 때 디바이스 드라이버를 설치할 수 있습니다.  

- [Auto Apply Drivers](/sccm/osd/understand/task-sequence-steps#BKMK_AutoApplyDrivers): 운영 체제 배포 과정에서 디바이스 드라이버를 자동으로 일치시키고 설치할 수 있습니다. 작업 순서 단계에서 검색된 각 하드웨어 디바이스에 가장 적합한 드라이버만 설치하도록 구성할 수 있습니다. 또는 해당 단계에서 검색된 각 하드웨어 디바이스와 호환되는 모든 드라이버를 설치하고 Windows 설치 프로그램에서 가장 적합한 드라이버를 선택하도록 지정합니다. 이 단계에 사용할 수 있는 드라이버를 제한하도록 드라이버 범주를 지정할 수도 있습니다.  

- [Apply Driver Package](/sccm/osd/understand/task-sequence-steps#BKMK_ApplyDriverPackage): 특정 드라이버 패키지의 모든 디바이스 드라이버를 Windows 설치에 대해 사용 가능하도록 설정할 수 있습니다. Windows 설치는 지정된 드라이버 패키지에서 필요한 디바이스 드라이버를 검색합니다. 독립 실행형 미디어를 만들 때 이 단계를 사용하여 디바이스 드라이버를 설치해야 합니다.  

이 작업 순서 단계를 사용할 때 OS를 배포할 컴퓨터에 드라이버를 설치하는 방식도 지정할 수 있습니다. 자세한 내용은 [작업을 자동화하는 작업 순서 관리](/sccm/osd/deploy-use/manage-task-sequences-to-automate-tasks)를 참조하세요.  



## <a name="BKMK_DriverReports"></a> 드라이버 보고서  

**드라이버 관리** 보고서 범주에 속한 몇 가지 보고서를 사용하여 드라이버 카탈로그의 디바이스 드라이버에 대한 일반적인 정보를 확인할 수 있습니다. 보고서에 대한 자세한 내용은 [보고](/sccm/core/servers/manage/reporting)를 참조하세요.
