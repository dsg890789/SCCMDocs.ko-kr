# <a name="planning-considerations-for-automating-tasks-in-configuration-manager"></a>Configuration Manager에서 작업을 자동화하기 위한 계획 고려 사항

*적용 대상: System Center Configuration Manager(현재 분기)*

 Configuration Manager 환경에서 작업을 자동화하는 작업 순서를 만들 수 있습니다. 이러한 작업은 참조 컴퓨터에서 OS를 캡처하는 것부터 하나 이상의 대상 컴퓨터에 OS를 배포하는 것까지 다양합니다. 작업 순서의 각 단계에는 작업 순서의 작업이 정의되어 있습니다. 작업 순서가 실행될 때 작업 순서는 각 단계의 작업을 로컬 시스템 컨텍스트의 명령줄 수준에서 수행합니다. 이것은 작업 순서가 사용자 개입 없이 완전히 자동화되어 실행됨을 의미합니다. 



##  <a name="BKMK_TSStepsActions"></a> 작업 순서 단계 및 작업  

 단계는 작업 순서의 기본 구성 요소입니다. 작업은 다음과 같은 명령을 포함할 수 있습니다.  
   - 참조 컴퓨터의 OS 구성 및 캡처합니다.  
   - 대상 컴퓨터에서 Windows, 하드웨어 드라이버, 구성 관리자 클라이언트 및 소프트웨어를 설치합니다.   


 단계의 작업은 작업 순서 단계의 명령을 정의합니다. 두 가지 유형의 작업이 있습니다.  
   - 명령줄 문자열을 사용하여 정의하는 작업은 *사용자 지정 작업*이라고 하며,  
   - Configuration Manager에서 사전에 정의하는 작업은 *기본 제공 작업*이라고 합니다.  


 작업 순서는 모든 조합의 사용자 지정 작업과 기본 제공 작업을 수행할 수 있습니다.  

 작업 순서 단계는 단계의 동작 방식을 제어하는 조건을 포함할 수 있습니다. 이러한 동작은 오류가 발생할 경우 작업 순서를 중지하거나 작업 순서를 계속하는 것이 포함됩니다. 조건의 한 가지 유형이 작업 순서 변수입니다. 예를 들어 이전 단계의 조건을 테스트하려면 **SMSTSLastActionRetCode** 변수를 사용하세요. 조건을 하나의 단계 또는 단계 그룹에 추가합니다.  

 작업 순서는 단계들을 순차적으로 처리합니다. 이 순서는 단계의 작업과 단계에 있는 모든 조건을 포함합니다. Configuration Manager에서 작업 순서 단계를 처리할 때 이전 단계가 완료되기 전까지 다음 단계는 시작되지 않습니다. 

 작업 순서는 다음의 경우 완료된 것으로 간주됩니다. 
   - 모든 단계가 완료되었습니다.  
   - 단계가 실패하면 Configuration Manager에서는 모든 단계가 완료되기 전에 작업 순서 실행을 중지합니다.  


 예를 들어 작업 순서의 단계가 배포 지점에서 참조된 이미지 또는 패키지를 찾을 수 없는 경우 작업 순서는 손상된 참조를 포함합니다. Configuration Manager는 실패한 단계에 오류 발생 시 계속되는 조건이 없는 경우 해당 지점에서 작업 순서 실행을 중지합니다.  

 > [!IMPORTANT]  
 >  기본적으로 한 단계 또는 작업이 실패하면 작업 순서가 실패합니다. 단계에 오류가 난 경우에도 작업 순서를 계속하려면 작업 순서를 편집하고 **옵션** 탭을 클릭한 다음 **오류 발생 시 계속**를 참조하세요.  

 작업 순서에 추가할 수 있는 단계에 대한 자세한 내용은 [작업 순서 단계](/sccm/osd/understand/task-sequence-steps)를 참조하세요.  



##  <a name="BKMK_TSGroups"></a> 작업 순서 그룹  

 작업 순서 내의 여러 단계를 그룹화할 수 있습니다. 작업 순서 그룹은 이름, 선택적 설명 및 선택적 조건으로 구성됩니다. 작업 순서는 다음 단계를 진행하기 전에 그룹 조건을 한 단위로 평가합니다. 그룹을 서로 중첩하거나 여러 단계와 하위 그룹을 혼합하여 포함할 수 있습니다. 그룹은 공통 조건을 갖는 여러 단계를 결합하는 데 유용합니다.  

 작업 순서 그룹에 이름을 지정하세요. 고유하지 않아도 됩니다. 또한 작업 순서 그룹에 대한 선택적 설명을 제공할 수 있습니다.  

 > [!IMPORTANT]  
 >  기본적으로 그룹 내 단계 또는 포함된 그룹이 실패하면 작업 순서 그룹이 실패합니다. 단계나 포함된 그룹이 실패한 경우에도 작업 순서를 계속하려면 해당 단계 또는 그룹에 **오류 발생 시 계속** 옵션을 설정합니다.  

 다음 표에서는 단계를 그룹화할 경우 **오류 발생 시 계속** 옵션이 어떻게 작동하는지를 보여 줍니다.  

 이 예에서는 두 개의 작업 순서 그룹이 있고, 이 그룹들은 각각 3개의 작업 순서 단계를 포함하고 있습니다.  

 |작업 순서 그룹 또는 단계|오류 발생 시 계속 설정|  
 |---------------------------------|-------------------------------|  
 |**작업 순서 그룹 1**|**오류 발생 시 계속** 이 선택됨|  
 |작업 순서 단계 1|**오류 발생 시 계속** 이 선택됨|  
 |작업 순서 단계 2|설정되지 않음|  
 |작업 순서 단계 3|설정되지 않음|  
 |**작업 순서 그룹 2**|설정되지 않음|  
 |작업 순서 단계 4|설정되지 않음|  
 |작업 순서 단계 5|설정되지 않음|  
 |작업 순서 단계 6|설정되지 않음|  


 -   작업 순서 단계 1이 실패하면 작업 순서 단계 2로 작업 순서가 계속됩니다.  

 -   작업 순서 단계 2가 실패하면 작업 순서는 작업 순서 단계 3을 실행하지 않습니다. 작업 순서 그룹 1이 **오류 발생 시 계속**으로 구성되어 있으므로, 작업 순서는 작업 순서 그룹 2로 계속됩니다. 작업 순서는 다음으로 작업 순서 단계 4를 실행합니다.  

 -   작업 순서 단계 4가 실패하면 더 이상 단계가 실행되지 않습니다. 작업 순서 그룹 2에 대해 **오류 발생 시 계속** 설정이 구성되어 있지 않았기 때문에 작업 순서가 실패합니다.

##  <a name="BKMK_TSVariables"></a> 작업 순서 변수  

 작업 순서 변수는 이름과 값의 세트로서, 작업 순서 변수는 구성 관리자 클라이언트에서 컴퓨터, OS 및 사용자 상태 구성 작업의 구성 및 OS 배포 설정을 제공합니다. 작업 순서 변수는 작업 순서의 단계를 구성 및 사용자 지정하는 메커니즘을 제공합니다.  

 작업 순서를 실행하면 작업 순서는 다수의 작업 순서 설정을 환경 변수로 저장합니다. 기본 제공되는 작업 순서 변수의 값을 액세스 또는 변경할 수 있습니다. 또한 새 작업 순서 변수를 만들어 작업 순서가 대상 컴퓨터에 대해 실행되는 방식을 사용자 지정할 수도 있습니다.  

 다음 작업을 수행하려면 작업 순서 변수를 사용하세요.  

 -   작업 순서 작업에 대한 설정 구성  

 -   작업 순서 단계에 명령줄 인수 제공  

 -   작업 순서 단계 또는 그룹의 실행 여부를 결정하는 조건 평가  

 -   작업 순서에 사용되는 사용자 지정 스크립트에 값 제공  


 예를 들어 **도메인 또는 작업 그룹 가입** 작업 순서 단계가 포함된 작업 순서가 있는 경우, 작업 순서를 다양한 컬렉션에 배포할 수 있습니다. 여기서 컬렉션의 멤버 자격은 도메인 멤버 자격으로 결정됩니다. 각 컬렉션의 도메인 이름에 대한 컬렉션별 작업 순서 변수를 지정하세요. 그런 다음 해당 작업 순서 변수를 사용하여 작업 순서에서 적절한 도메인 이름을 제공합니다.  

 자세한 내용은 [How to use task sequence variables](/sccm/osd/understand/using-task-sequence-variables)\(작업 순서 변수 사용 방법\)을 참조하세요.



##  <a name="BKMK_TSCreate"></a> 작업 순서 만들기  

 작업 순서 만들기 마법사를 사용하여 작업 순서를 만듭니다. 이 마법사를 통해, 특정 작업을 수행하는 기본 제공 작업 순서 또는 여러 다양한 작업을 수행할 수 있는 사용자 지정 작업 순서를 만들 수 있습니다. 마법사를 사용하면 다음 유형의 작업 순서를 만들 수 있습니다.

 - 대상 컴퓨터에 기존 OS 이미지 설치  

 - 참조 컴퓨터의 OS 이미지 빌드 및 캡처  

 - 대상 컴퓨터의 OS 업그레이드 패키지에서 Windows 10으로 업그레이드   

 - 사용자 지정된 작업 또는 특수한 OS 배포를 수행하는 사용자 지정 작업 순서 만들기  


 자세한 내용은 [작업 순서 만들기](/sccm/osd/deploy-use/manage-task-sequences-to-automate-tasks#BKMK_CreateTaskSequence)를 참조하세요.  



##  <a name="BKMK_TSEdit"></a> 작업 순서 편집  

 **작업 순서 편집기**를 사용하여 작업 순서를 편집하세요. 이 편집기를 사용하면 작업 순서에서 다음과 같은 변경 작업을 수행할 수 있습니다.  

 - 작업 순서에서 단계 추가 또는 제거  

 - 작업 순서 내 단계 순서 변경  

 - 단계 그룹 추가 또는 제거  

 - 오류 발생 시 작업 순서가 계속 실행될지 여부 지정  

 - 작업 순서의 단계 및 그룹에 조건 추가  


 > [!IMPORTANT]  
 >  편집 결과로 개체에 대한 연결되지 않은 참조가 작업 순서에 있을 경우 편집기를 닫으려면 그 전에 참조를 수정해야 합니다. 가능한 작업은 다음과 같습니다.  
 > - 참조 수정  
 > - 작업 순서에서 참조되지 않는 개체 삭제  
 > - 끊어진 참조를 수정 또는 제거하기 전까지 실패한 작업 순서 단계를 일시적으로 사용하지 않도록 설정  


 작업 순서를 편집하는 방법에 대한 자세한 내용은 [작업 순서 편집](/sccm/osd/deploy-use/manage-task-sequences-to-automate-tasks#BKMK_ModifyTaskSequence)을 참조하세요.  



##  <a name="BKMK_TSDeploy"></a> 작업 순서 배포  

 작업 순서를 구성 관리자 컬렉션에 있는 대상 컴퓨터에 배포하세요. 알 수 없는 컴퓨터에 운영 체제를 배포하는 데에는 기본 제공되는 **모든 알 수 없는 컴퓨터** 컬렉션을 사용합니다. 사용자 컬렉션에는 작업 순서를 배포할 수 없습니다.  

 > [!IMPORTANT]  
 >  부적절한 컬렉션에 운영 체제를 설치하는 작업 순서는 배포하지 마세요. 작업 순서를 배포하는 컬렉션은 OS를 설치할 해당 컴퓨터만 포함해야 합니다. 원치 않는 OS 배포를 방지하려면 위험이 큰 배포에 대한 설정을 구성합니다. 자세한 내용은 [높은 위험 수준의 배포를 관리하는 설정](/sccm/core/servers/manage/settings-to-manage-high-risk-deployments)을 참조하세요.  

 작업 순서를 수신하는 각 대상 컴퓨터에서는 배포에 지정된 설정에 따라 작업 순서를 실행합니다. 작업 순서 자체에는 연결된 파일 또는 프로그램이 포함되어 있지 않습니다. 작업 순서가 참조하는 모든 파일은 이미 대상 컴퓨터에 존재하거나 클라이언트가 액세스할 수 있는 배포 지점에 상주해야 합니다. 

 > [!NOTE]  
 > 작업 순서는 프로그램이 참조하는 패키지를 설치하며 이는 해당 프로그램 또는 패키지가 대상 컴퓨터에 이미 설치되어 있더라도 마찬가지입니다. 
 > 
 > 작업 순서에서 응용 프로그램을 설치하는 경우 해당 응용 프로그램에 대한 요구 사항 규칙이 충족되고 응용 프로그램이 아직 설치되지 않은 경우에만 응용 프로그램이 설치됩니다. 이때 이 응용 프로그램에 대해 지정된 검색 방법이 사용됩니다.  

 Configuration Manager 클라이언트는 클라이언트 정책을 다운로드할 때 작업 순서 배포를 실행합니다. 다음 폴링 주기를 기다리지 않고 이 작업을 트리거하려면 [구성 관리자 클라이언트에 대한 정책 검색 시작](/sccm/core/clients/manage/manage-clients#BKMK_PolicyRetrieval)을 참조하세요.  

 쓰기 필터를 사용하는 Windows Embedded 장치에 작업 순서를 배포할 경우 배포 중 장치에서 쓰기 필터를 사용하지 않도록 설정한 다음 배포 후 장치를 다시 시작할지 여부를 지정할 수 있습니다. 쓰기 필터가 사용하도록 설정된 경우 작업 순서는 임시 오버레이에 배포되고 장치가 다시 시작될 때 이 작업 순서는 사용할 수 없습니다.  

 > [!NOTE]  
 >  작업 순서를 Windows Embedded 장치에 배포할 경우 해당 장치가 유지 관리 기간이 구성된 컬렉션의 구성원인지 확인하세요. 이에 따라 쓰기 필터의 비활성 및 활성 시점과 장치가 다시 시작하는 시점을 관리할 수 있습니다.  
 >   
 >  클라이언트에서 유지 관리 외 기간에 작업 순서를 다운로드하면 이 작업 순서는 두 번 다운로드됩니다. 이 시나리오에서는 클라이언트가 작업 순서를 다운로드하고, 쓰기 필터를 사용하지 않도록 설정하고, 컴퓨터를 다시 시작한 다음, 작업 순서를 다시 다운로드합니다. 이것은 처음에 작업 순서가 장치가 다시 시작될 때 지워지는 임시 오버레이에 다운로드되었기 때문입니다.  

 작업 순서를 배포하는 방법에 대한 자세한 내용은 [작업 순서 배포](/sccm/osd/deploy-use/manage-task-sequences-to-automate-tasks#BKMK_DeployTS)를 참조하세요.  



##  <a name="BKMK_TSExportImport"></a> 작업 순서 내보내기 및 가져오기  

 Configuration Manager를 사용하여 작업 순서를 내보내고 가져올 수 있습니다. 작업 순서를 내보낼 때 작업 순서에서 참조된 개체를 포함할 수 있습니다. 

 자세한 내용은 [작업 순서 내보내기 및 가져오기](/sccm/osd/deploy-use/manage-task-sequences-to-automate-tasks#BKMK_ExportImport)를 참조하세요.  



##  <a name="BKMK_TSRun"></a> 작업 순서 실행  

 작업 순서는 항상 로컬 시스템 계정으로 실행됩니다. 작업 순서가 실행되면 구성 관리자 클라이언트는 작업 순서의 단계를 시작하기 전에 먼저 참조된 패키지가 있는지 확인합니다. 유효성 검사를 수행할 수 없거나 다운로드할 수 없는 참조된 패키지의 경우, 작업 순서는 연결된 작업 순서 단계에 대한 오류를 반환합니다.  

 > [!Note]  
 > **명령줄 실행** 작업 순서 단계에서는 명령을 다른 계정으로 실행할 수 있습니다.  

 다운로드하여 실행할 작업 순서 배포를 구성하는 경우 구성 관리자 클라이언트에서는 모든 종속 콘텐츠를 캐시에 다운로드합니다. 클라이언트 캐시 크기가 너무 작거나 콘텐츠를 찾을 수 없으면 작업 순서는 실패합니다. 클라이언트가 상태 메시지를 생성합니다. 

 필수인 경우에만 클라이언트가 콘텐츠를 다운로드하도록 지정할 수도 있습니다. 이 작업을 수행하려면 작업 순서 배포에서 **작업 순서를 실행하기 전에 콘텐츠를 로컬에 다운로드**를 선택합니다. 또 다른 옵션은 **배포 지점에서 프로그램 실행**을 수행하는 것입니다. 이 옵션이 있으므로 클라이언트는 파일을 먼저 캐시에 다운로드하지 않고 배포 지점에서 바로 설치할 수 있습니다. 

 작업 순서 배포를 **사용 가능**으로 구성하면, 클라이언트가 작업 순서에 대한 종속 콘텐츠를 찾을 수 없는 경우 즉시 오류를 보냅니다. **필수** 배포의 경우 이 상황에서 구성 관리자 클라이언트는 대기했다가, 클라이언트가 액세스할 수 있는 콘텐츠 위치에 콘텐츠가 아직 복제되지 않는 경우 최종 기한 전까지 콘텐츠의 다운로드를 다시 시도합니다.  

 작업 순서가 성공적으로 완료되거나 실패하는 경우 Configuration Manager는 클라이언트 기록에 이 상태를 기록합니다. 

 작업 순서가 컴퓨터에서 시작된 후에는 이를 취소하거나 중지할 수 없습니다.  

 > [!IMPORTANT]  
 >  작업 순서 단계에 따라 컴퓨터를 다시 시작해야 하는 경우 클라이언트는 포맷된 디스크 파티션으로 부팅할 수 있어야 합니다. 그렇지 않으면 작업 순서에서 사용자가 지정한 오류 처리 방식에 상관없이 작업 순서가 실패합니다.  

 작업 순서의 종속적 개체가 더 최신 버전으로 업데이트되는 경우 패키지를 참조하는 모든 작업 순서는 자동으로 업데이트되고, 이전에 배포한 업데이트 수에 상관없이 가장 최신 버전을 참조하게 됩니다.  



##  <a name="BKMK_TSMaintenanceWindow"></a> 유지 관리 기간을 사용하여 작업 순서 실행 가능 시기 지정  

 장치 컬렉션의 유지 관리 기간을 정의하여 작업 순서가 실행될 수 있는 시기를 지정할 수 있습니다. 시작 날짜, 시작/종료 시간 및 되풀이 패턴으로 유지 관리 기간을 구성합니다. 유지 관리 기간의 일정을 설정하는 경우 유지 관리 기간이 작업 순서에만 적용되도록 지정할 수 있습니다. 자세한 내용은 [유지 관리 기간을 사용하는 방법](/sccm/core/clients/manage/collections/use-maintenance-windows)을 참조하세요.  

 > [!IMPORTANT]  
 >  작업 순서를 실행할 유지 관리 기간을 구성하는 경우 유지 관리 기간이 종료되는 경우라도 이미 시작된 작업 순서는 계속 실행됩니다.  



##  <a name="BKMK_TSNetworkAccessAccount"></a> 작업 순서와 네트워크 액세스 계정  

 로컬 시스템 계정에서만 작업 순서를 실행할 수 있지만 다음과 같은 경우에는 [네트워크 액세스 계정](/sccm/core/plan-design/hierarchy/accounts#network-access)을 구성해야 합니다.  

 - 작업 순서가 배포 지점에서 Configuration Manager 콘텐츠에 액세스하려고 시도하는 경우. 네트워크 액세스 계정을 올바로 구성하세요. 그렇지 않으면, 작업 순서가 실패합니다.   

 - 부팅 이미지를 사용하여 OS 배포를 시작하는 경우. 이 경우 Configuration Manager는 전체 OS가 아닌 Windows PE 환경을 사용합니다. Windows PE 환경은 어떤 도메인에도 속하지 않는 자동으로 생성된 무작위 이름을 사용합니다. 네트워크 액세스 계정을 올바로 구성하지 않으면, 컴퓨터에서 작업 순서에 필요한 콘텐츠를 액세스할 수 없습니다.  


 > [!NOTE]  
 >  네트워크 액세스 계정은 프로그램을 실행하거나 응용 프로그램을 설치하거나 업데이트를 설치하거나 작업 순서를 실행하기 위한 보안 컨텍스트로 절대 사용되지 않습니다. 네트워크 액세스 계정은 네트워크에 있는 연결된 리소스에 액세스하는 데에만 사용합니다.  


 네트워크 액세스 계정에 대한 자세한 내용은 [네트워크 액세스 계정](/sccm/core/plan-design/hierarchy/manage-accounts-to-access-content#bkmk_NAA)을 참조하세요.  



##  <a name="BKMK_TSCreateMedia"></a> 작업 순서 미디어 만들기  

 작업 순서와 관련 파일 및 종속성을 여러 유형의 미디어에 기록할 수 있습니다. Configuration Manager은 캡처용 DVD나 USB 플래시 드라이브와 같은 이동식 미디어, 독립 실행형 및 부팅 가능한 미디어를 지원합니다. 미리 준비된 미디어에서는 Windows 이미지(WIM) 파일을 사용합니다.  

 미디어를 작성할 때에는 액세스를 제어할 암호를 지정하세요. 그러면 사용자는 작업 순서를 실행할 대상 컴퓨터에서 암호를 입력해야 합니다.  

 미디어에서 작업 순서를 실행할 때에는 미디어의 지정된 프로세서 아키텍처가 인식되지 않습니다. 지정된 아키텍처가 대상 컴퓨터와 일치하지 않는 경우, 작업 순서는 여전히 실행하려고 시도합니다. 미디어의 아키텍처가 대상 컴퓨터의 아키텍처와 일치하지 않는 경우에는, 작업 순서가 실패합니다.  

 자세한 내용은 [작업 순서 미디어 만들기](/sccm/osd/deploy-use/create-task-sequence-media)를 참조하세요.  


### <a name="media-types"></a>미디어 유형
 Configuration Manager에서 지원하는 미디어 유형은 다음과 같습니다.  

#### <a name="capture-media"></a>미디어 캡처
 미디어 캡처는 사용자가 Configuration Manager 인프라의 외부에서 구성하고 만드는 OS 이미지를 캡처합니다. 미디어 캡처에는 작업 순서 실행 전에 실행할 수 있는 사용자 지정 프로그램이 포함될 수 있습니다. 사용자 지정 프로그램은 데스크톱과 상호 작용하거나, 사용자에게 입력 값을 묻거나, 작업 순서에 사용할 변수를 생성할 수 있습니다.  

 자세한 내용은 [미디어 캡처 만들기](/sccm/osd/deploy-use/create-capture-media)를 참조하세요.  

#### <a name="stand-alone-media"></a>독립 실행형 미디어
 독립 실행형 미디어에는 작업 순서와, 작업 순서를 실행하는 데 필요한 모든 관련 개체가 포함되어 있습니다. 독립 실행형 미디어 작업 순서는 Configuration Manager가 네트워크에 연결되어 있지 않거나 연결이 제한적인 경우에 실행할 수 있습니다. 독립 실행형 미디어는 다음 방식으로 실행합니다.  

 - 대상 컴퓨터가 부팅되지 않는 경우 작업 순서와 연결된 Windows PE 이미지가 독립 실행형 미디어에서 사용되어 작업 순서가 시작됩니다.  

 - 독립 실행형 미디어를 수동으로 시작합니다. 사용자가 컴퓨터에 로그인되어 있으면 미디어에서 작업 순서를 시작할 수 있습니다.  


 > [!IMPORTANT]  
 >  독립 실행형 미디어 작업 순서의 단계는 네트워크에서 데이터를 검색하지 않고 실행될 수 있어야 합니다. 그렇지 않으면 데이터를 검색하는 작업 순서 단계가 실패합니다. 예를 들어 패키지를 가져오는 배포 지점이 필요한 작업 순서 단계는 실패합니다. 독립 실행형 미디어가 필요한 패키지를 포함하는 경우에는 작업 순서 단계가 성공합니다.  


 자세한 내용은 [독립 실행형 미디어 만들기](/sccm/osd/deploy-use/create-stand-alone-media)를 참조하세요.  

#### <a name="bootable-media"></a>부팅 가능한 미디어
 부팅 가능한 미디어에는 Configuration Manager 인프라에 연결할 수 있도록 대상 컴퓨터를 시작하는 데 필요한 파일이 포함되어 있습니다. 따라서 컬렉션의 멤버 자격을 기반으로 하여 실행할 작업 순서를 결정합니다. 이 미디어는 작업 순서 또는 종속 개체를 포함하지 않습니다. 대신 클라이언트가 네트워크를 통해 콘텐츠를 다운로드합니다. 이 방법은 새 컴퓨터나 OS가 대상 컴퓨터에 없는 운영 체제 미설치 배포에 유용합니다.  

 자세한 내용은 [부팅 가능한 미디어 만들기](/sccm/osd/deploy-use/create-bootable-media)를 참조하세요.  

#### <a name="prestaged-media"></a>미리 준비된 미디어
 미리 준비된 미디어는 프로비전되지 않은 대상 컴퓨터에 OS 이미지를 배포하며, Windows 이미지(WIM) 파일로 저장됩니다. 이 파일은 제조업체에서 또는 엔터프라이즈 준비 센터에서 운영 체제 미설치 컴퓨터에 설치할 수 있습니다. 미리 준비된 미디어의 이점은 이러한 곳에서 Configuration Manager 환경에 연결할 필요가 없다는 것입니다.  

 자세한 내용은 [사전 준비된 미디어 만들기](/sccm/osd/deploy-use/create-prestaged-media)를 참조하세요.
