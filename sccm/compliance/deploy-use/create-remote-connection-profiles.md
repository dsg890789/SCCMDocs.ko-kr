# <a name="remote-connection-profiles-in-configuration-manager"></a>Configuration Manager의 원격 연결 프로필

*적용 대상: Configuration Manager(현재 분기)*

Configuration Manager 원격 연결 프로필을 사용하여 사용자가 작업 컴퓨터에 원격으로 연결할 수 있습니다. 이러한 프로필을 통해 계층의 사용자에게 원격 데스크톱 연결을 설정을 배포할 수 있습니다. 사용자는 VPN 연결을 통해 원격 데스크톱을 통해 기본 작업 컴퓨터에 액세스할 수 있습니다.  

> [!IMPORTANT]  
> Configuration Manager를 사용하여 원격 연결 프로필 설정을 지정하는 경우 클라이언트는 Windows 로컬 정책에 설정을 저장합니다. 이러한 설정은 다른 애플리케이션에서 구성한 원격 데스크톱 설정을 재정의할 수 있습니다. 또한 Windows 그룹 정책을 사용하여 원격 데스크톱 설정을 구성하면 그룹 정책에 지정된 설정이 Configuration Manager 설정을 재정의합니다.

Configuration Manager는 클라이언트, **원격 PC 연결**에 보안 그룹을 만듭니다. 원격 연결 프로필을 배포하는 경우 클라이언트는 컴퓨터의 기본 사용자를 이 그룹에 추가합니다. 로컬 관리자는 이 그룹에 사용자를 수동으로 추가하거나 제거할 수 있지만, Configuration Manager가 다음에 프로필의 규정 준수를 평가할 때 멤버 자격을 업데이트합니다.

> [!IMPORTANT]  
> 사용자와 디바이스 간의 사용자 디바이스 선호도 관계가 변경되는 경우 Configuration Manager에서 원격 연결 프로필 및 Windows 방화벽 설정을 사용하지 않도록 설정하여 해당 컴퓨터에 연결할 수 없게 됩니다.

## <a name="prerequisites"></a>전제 조건  

### <a name="external-dependencies"></a>외부 종속성  

- 사용자가 인터넷에서 연결할 수 있도록 하려면 원격 데스크톱 게이트웨이 서버를 설치하고 구성해야 합니다. 원격 데스크톱 게이트웨이 서버를 설치하고 구성하는 방법에 대한 자세한 내용은 [원격 데스크톱 서비스 - 어디에서나 액세스](https://docs.microsoft.com/windows-server/remote/remote-desktop-services/rds-plan-access-from-anywhere)를 참조하세요.

- 클라이언트에서 호스트 기반 방화벽을 실행하는 경우 mstsc.exe 프로그램을 허용해야 합니다. 원격 연결 프로필을 구성할 때 **Windows 도메인 및 프라이빗 네트워크에서 연결에 대한 Windows 방화벽 예외 허용**을 사용하도록 설정해야 합니다. 이 설정을 통해 Configuration Manager에서 자동으로 Windows 방화벽을 구성할 수 있습니다.

    > [!TIP]
    > Windows 방화벽을 구성하는 그룹 정책 설정은 Configuration Manager에서 설정하는 구성을 재정의할 수 있습니다. 그룹 정책을 사용하여 Windows 방화벽을 구성하는 경우 그룹 정책 설정이 mstsc를 차단하지 않는지 확인합니다.

    클라이언트에서 다른 호스트 기반 방화벽을 실행하는 경우에는 이 방화벽 종속성을 수동으로 구성하세요.  

### <a name="configuration-manager-dependencies"></a>Configuration Manager 종속성  

- 사용자가 작업 컴퓨터에 연결할 수 있으려면 해당 컴퓨터가 사용자의 기본 디바이스여야 합니다. 자세한 내용은 [사용자 디바이스 선호도를 사용하여 사용자와 디바이스 연결](/configmgr/apps/deploy-use/link-users-and-devices-with-user-device-affinity)을 참조하세요.

- 원격 연결 프로필을 관리하려면 사용자 계정에 Configuration Manager의 특정 사용 권한이 있어야 합니다. **준수 설정 관리자** 기본 제공 역할에는 이러한 프로필을 관리하는 데 필요한 권한이 포함됩니다. 자세한 내용은 [역할 기반 관리 구성](/configmgr/core/servers/deploy/configure/configure-role-based-administration)을 참조하세요.

## <a name="security-and-privacy-considerations"></a>보안 및 개인 정보 고려 사항

### <a name="security-considerations"></a>보안 고려 사항  

- 사용자에게 기본 디바이스를 식별할 수 있도록 허용하는 대신 사용자 디바이스 선호도를 수동으로 지정합니다. 사용량 기준 구성은 사용하도록 설정하지 마세요.

  - 원격 연결 프로필을 배포하려면 먼저 **작업 컴퓨터의 모든 기본 사용자가 원격으로 연결할 수 있도록 허용**하는 옵션을 사용하도록 설정해야 합니다. 이 구성을 사용하는 경우 항상 사용자 디바이스 선호도를 수동으로 지정해야 합니다. 사용자가 수집하거나 디바이스에서 Configuration Manager 수집하는 정보를 신뢰할 수 있는 정보로 간주해서는 안 됩니다. 프로필을 배포하는데 신뢰할 수 있는 관리자가 사용자 디바이스 선호도를 지정하지 않으면 권한 없는 사용자가 상승된 권한을 받고 컴퓨터에 원격으로 연결할 수 있습니다.

  - Configuration Manager는 상태 메시지를 통해 사용 현황 기반 정보를 수집하며, 이는 빠르지만 안전하지 않은 통신 채널입니다. 이러한 위협을 완화하려면 클라이언트 컴퓨터와 관리 지점 간에 SMB(서버 메시지 블록) 서명 또는 IPsec(인터넷 프로토콜 보안)을 사용하세요.

- 사이트 서버 컴퓨터에 대한 로컬 관리자 권한 제한 사이트 서버에 대한 로컬 관리자는 Configuration Manager에서 자동으로 만들고 유지 관리하는 **Remote PC Connect** 보안 그룹에 수동으로 구성원을 추가할 수 있습니다. 이 작업을 하면 구성원이 원격 데스크톱 권한을 받게 되므로 권한 상승이 발생할 수 있습니다.

### <a name="privacy-considerations"></a>개인 정보 고려 사항  

사용자가 작업 컴퓨터에 원격으로 연결하는 경우 .wsrdp 파일을 다운로드합니다. 이 파일에는 디바이스 이름과 원격 데스크톱 게이트웨이 서버 이름이 포함됩니다. 이러한 값은 원격 데스크톱 세션을 만드는 데 필요합니다. .wsrdp 파일은 다운로드되어 자동으로 로컬에 저장됩니다. 다음에 사용자가 원격 데스크톱 세션을 실행할 때 이 파일을 덮어쓰게 됩니다.  

## <a name="create-a-profile"></a>프로필 만들기

1. Configuration Manager 콘솔에서 **자산 및 준수** 작업 영역으로 이동하여 **준수 설정**을 확장하고 **원격 연결 프로필**을 선택합니다.  

1. 리본의 **홈** 탭에 있는 **만들기** 그룹에서 **원격 연결 프로필 만들기**를 선택합니다.  

1. **원격 연결 프로필 만들기 마법사**의 **일반** 페이지에서 프로필에 대한 이름 및 선택적 설명을 지정합니다. 두 값에는 최대 256자까지 입력합니다.  

1. **프로필 설정** 페이지에서 다음 설정을 지정합니다.  

    - **원격 데스크톱 게이트웨이 서버의 전체 이름 및 포트(옵션)** : 연결에 사용할 원격 데스크톱 게이트웨이 서버의 이름을 지정합니다. 이 값에는 다음과 같은 요구 사항이 있습니다.

        - 서버 이름은 256자를 초과할 수 없습니다.
        - 대문자, 소문자 및 숫자를 포함할 수 있습니다.
        - 세그먼트 간의 마침표(`.`)와 포트 앞에 콜론(`:`) 외에 유일한 특수 문자는 대시(`–`) 및 밑줄(`_`)입니다.
        - Configuration Manager는 이 값에 다국어 도메인 이름을 사용하는 것을 지원하지 않습니다.

    - **네트워크 수준 인증을 사용하여 원격 데스크톱을 실행하는 컴퓨터에서만 연결 허용**: 기본적으로 사용하도록 설정되면 이 설정은 연결에 대한 보안 수준을 추가합니다. 자세한 내용은 [원격 데스크톱 액세스 권한 부여](https://docs.microsoft.com/windows-server/remote/remote-desktop-services/clients/remote-desktop-allow-access#why-allow-connections-only-with-network-level-authentication)를 참조하세요.

    - 다음 연결 설정을 사용하도록 설정합니다.

        - **작업 컴퓨터에 원격 연결 허용**

        - **작업 컴퓨터의 모든 기본 사용자가 원격으로 연결하도록 허용**

        - **Windows 도메인 및 프라이빗 네트워크에서 연결에 대한 Windows 방화벽 예외 허용**

        > [!IMPORTANT]  
        > 계속하려면 세 가지 설정이 모두 동일해야 합니다.

        원격 연결을 해제하기 위해 프로필을 배포할 때만 이러한 설정을 사용하지 않습니다.

1. 마법사를 완료합니다.

**자산 및 규정 준수** 작업 영역의 **원격 연결 프로필** 노드에 새 프로필이 표시됩니다.  

## <a name="deploy"></a>배포

1. Configuration Manager 콘솔에서 **자산 및 준수** 작업 영역으로 이동하여 **준수 설정**을 확장하고 **원격 연결 프로필**을 선택합니다.

1. **원격 연결 프로필** 목록에서 배포하려는 프로필을 선택합니다. 리본의 **홈** 탭에 있는 **배포** 그룹에서 **배포**를 선택합니다.  

1. **원격 연결 프로필 배포** 창에서 다음 정보를 지정합니다.

    - **컬렉션**: 프로필을 배포하려는 디바이스 컬렉션을 찾아서 선택합니다.

    - **지원되는 경우 비규정 준수 규칙 재구성**: 디바이스에서 정책을 준수하지 않을 때 프로필 설정이 자동으로 재구성되도록 하려면 이 설정을 사용하도록 설정합니다. 프로필이 존재하지 않는 경우 비규격일 수 있습니다.

    - **유지 관리 기간을 벗어나도 재구성 허용**: 프로필을 배포할 컬렉션에 대해 유지 관리 기간을 구성하는 경우 이 옵션을 사용하도록 설정하여 유지 관리 기간 외에 Configuration Manager가 재구성하도록 할 수 있습니다. 자세한 내용은 [유지 관리 기간을 사용하는 방법](/configmgr/core/clients/manage/collections/use-maintenance-windows)을 참조하세요.

    - **경고 생성**: 준수 경고를 구성하려면 이 옵션을 사용하도록 설정합니다.

    - **이 구성 기준에 대한 준수 평가 일정 지정**: 클라이언트가 프로필을 평가하는 단순 또는 사용자 지정 일정을 지정합니다.

1. **확인**을 선택하여 창을 닫고 배포를 만듭니다.

### <a name="client-evaluation"></a>클라이언트 평가

클라이언트는 사용자가 로그인할 때 프로필을 평가합니다.

디바이스가 원격 연결 프로필을 배포하는 컬렉션을 벗어나면 Configuration Manager가 디바이스에서 설정을 사용하지 않도록 설정합니다. 그러나 이런 프로세스가 제대로 적용되려면 사이트에서 구성 항목이나 구성 항목이 포함된 구성 기준을 이미 하나 이상 배포했어야 합니다.

### <a name="conflict-resolution"></a>충돌 해결

설정이 충돌하는 둘 이상의 원격 연결 프로필을 동일한 디바이스에 배포하지 마세요. 예를 들어 다른 설정의 두 프로필을 동일한 컬렉션에 배포할 수 있습니다. **지원되는 경우 비규격 규칙을 재구성**하는 프로필 배포를 하나만 구성하면 됩니다. 이 배포는 다른 프로필의 설정을 재정의할 수 있습니다. Configuration Manager는 이러한 유형의 원격 연결 프로필 배포를 지원하지 않습니다.

## <a name="monitor"></a>모니터

Configuration Manager 콘솔에서 **모니터링** 작업 영역으로 이동하고 **배포**를 선택합니다. **배포** 목록에서 원격 연결 프로필 배포를 선택합니다.

기본 페이지에서 원격 연결 프로필 배포의 호환성에 대한 요약 정보를 검토할 수 있습니다. 자세한 정보를 보려면 프로필 배포를 선택합니다. 그런 다음 리본의 **홈** 탭에 있는 **배포** 그룹에서 **상태 보기**를 선택합니다. 이 작업을 수행하면 **배포 상태** 페이지가 열립니다.  

**배포 상태** 페이지에는 다음 탭이 있습니다.  

- **규격**: 영향받는 자산 수에 따라 원격 연결 프로필의 호환성을 표시합니다.

    > [!IMPORTANT]  
    > 클라이언트는 해당하지 않는 경우 원격 연결 프로필을 평가하지 않습니다. 그러나 여전히 규정 준수를 보고합니다.

- **오류**: 영향받는 자산 수에 따라 선택한 원격 연결 프로필 배포에 대한 모든 오류 목록을 표시합니다.

- **비준수**: 영향받는 자산 수에 따라 원격 연결 프로필 내에 있는 모든 비호환 규칙 목록을 표시합니다.

- **알 수 없음**: 선택한 원격 연결 프로필 배포의 규정 준수를 보고하지 않은 모든 디바이스의 목록과 디바이스의 현재 클라이언트 상태를 표시합니다.

어느 탭에서든 규칙을 열어서 **자산 및 준수** 작업 영역의 **사용자** 노드 아래에 임시 노드를 만들 수 있습니다. 이 하위 노드는 선택한 탭의 준수 상태와 함께 모든 디바이스를 포함합니다.

**자산 세부 정보** 창에 이 프로필에 대해 선택한 준수 상태와 디바이스가 표시됩니다. 목록에서 추가 정보를 표시할 디바이스를 엽니다.

## <a name="reports"></a>보고서

Configuration Manager에는 원격 연결 프로필에 대한 정보를 모니터링하는 데 사용할 수 있는 기본 제공 보고서가 포함됩니다. 이러한 보고서에는 **호환 및 설정 관리**라는 보고서 범주가 있습니다.

> 규정 준수 설정 보고서에서 **디바이스 필터** 및 **사용자 필터** 매개 변수를 사용할 경우 와일드카드(`%`) 문자를 사용해야 합니다.  

Configuration Manager에서 보고를 구성하는 방법에 대한 자세한 내용은 [보고](/configmgr/core/servers/manage/reporting)를 참조하세요.
