# <a name="how-to-deploy-to-pilot-with-desktop-analytics"></a>데스크톱 분석을 사용 하 여 파일럿에 배포 하는 방법

데스크톱 분석의 이점 중 하나는 가장 광범위 한 요소를 제공 하는 가장 작은 장치 집합을 식별 하는 데 도움을 주는 것입니다. Windows 업그레이드 및 업데이트의 파일럿에 가장 중요 한 요소를 중점적으로 설명 합니다. 파일럿이 더 성공 하도록 하면 프로덕션 환경에서 광범위 하 게 배포할 수 있습니다.  

[!INCLUDE [Definition of pilot and production](includes/define-pilot-prod.md)]


## <a name="identify-devices"></a>장치 식별

첫 번째 단계는 파일럿에 포함할 장치를 식별 하는 것입니다. 데스크톱 분석에서는 보고 된 데이터를 기반으로 하는 장치를 권장 하며,이 목록에는 장치를 포함 하거나 바꿀 수 있습니다.

1. [데스크톱 분석 포털로](https://aka.ms/desktopanalytics)이동 하 고, 관리 그룹에서 **배포 계획**을 선택 합니다.

1. 배포 계획을 선택 합니다.

1. 배포 계획 메뉴의 준비 그룹에서 **파일럿 식별**을 선택 합니다.

최적 적용 범위에 대해 포함 하 여 권장 되는 장치 수를 보여 주는 데스크톱 분석의 데이터가 표시 됩니다. 이 알고리즘은 주로 중요 한 앱과 중요 한 앱의 사용과 하드웨어 구성의 범위를 기반으로 합니다.

추가 권장 장치 목록에 대해 다음 작업을 수행 합니다.

- **파일럿에 모두 추가**: 모든 권장 장치를 파일럿 그룹에 추가 합니다.
- **파일럿에 추가**: 개별 장치만 추가
- 파일럿에서 특정 장치를 **바꿉니다** .
- 변경 작업이 완료 되 면 **다시 계산**

또한 파일럿에서 포함 하거나 제외할 Configuration Manager 컬렉션에 대 한 시스템 차원의 결정을 내릴 수 있습니다. 주 데스크톱 분석 메뉴의 전역 설정 그룹에서 **전역 파일럿**을 선택 합니다.

### <a name="example"></a>예

- Configuration Manager에서 데스크톱 분석 연결을 구성 하 여 **모든 시스템** 컬렉션을 대상으로 합니다. 이 작업은 모든 클라이언트를 서비스에 등록 합니다.
- 데스크톱 분석과 동기화 할 추가 컬렉션도 구성 합니다.
    - 모든 Windows 10 클라이언트
    - 모든 IT 장치
    - CEO 사무실
- **전역 파일럿** 설정에는 **모든 IT 장치** 컬렉션을 포함 합니다. **CEO office** 컬렉션을 제외 합니다.
- 배포 계획을 만들고 **모든 Windows 10 클라이언트** 컬렉션을 **대상 그룹**으로 선택 합니다.
- **파일럿 장치 포함** 목록에는 **대상 그룹**의 장치 하위 집합이 포함 됩니다. 전체 파일럿 *포함* 목록에도 있는 **모든 Windows 10 클라이언트** : **모든 IT 장치**  
- **추가 권장 장치** 목록에는 중요 한 자산에 대 한 최대 검사 및 중복성을 제공 하는 **대상 그룹** 의 장치 집합이 포함 되어 있습니다.  데스크톱 분석은이 목록에서 전 세계 파일럿 *제외* 목록의 모든 장치를 제외 합니다. **CEO office**


## <a name="address-issues"></a>문제 해결

데스크톱 분석 포털을 사용 하 여 배포를 차단할 수 있는 자산에 대해 보고 된 문제를 검토 합니다. 그런 다음 제안 된 수정 사항을 승인, 거부 또는 수정 합니다. 파일럿 배포를 시작 하기 전에 모든 항목에 대해 **준비** 됨 또는 **준비 (재구성 포함)** 로 표시 되어야 합니다.

1. [데스크톱 분석 포털로](https://aka.ms/desktopanalytics)이동 하 고, 관리 그룹에서 **배포 계획**을 선택 합니다.  

2. 배포 계획을 선택 합니다.  

3. 배포 계획 메뉴의 준비 그룹에서 **파일럿 준비**를 선택 합니다.  

4. **앱** 탭에서 입력이 필요한 앱을 검토 합니다.  

5. 각 앱에 대해 앱 이름을 선택 합니다. 정보 창에서 권장 사항을 검토 하 고 업그레이드 결정을 선택 합니다. **검토 안 함** 또는 불가능을 선택 **하는 경우**데스크톱 분석에는 파일럿 배포에이 앱이 있는 장치가 포함 되지 않습니다. **준비 (재구성 포함)** 를 선택 하는 경우 **재구성 정보** 를 사용 하 여 *다시 설치* 하거나 *제조업체의 권장 버전을 확인*하는 등의 문제를 해결 하기 위해 수행할 작업을 캡처할 수 있습니다.

6. 다른 자산에 대해이 검토를 반복 합니다.  


## <a name="create-software"></a>소프트웨어 만들기

Windows를 배포 하기 전에 먼저 Configuration Manager에서 소프트웨어 개체를 만듭니다. 자세한 내용은 [Windows 10 전체 업그레이드 작업 순서](https://docs.microsoft.com/sccm/osd/deploy-use/create-a-task-sequence-to-upgrade-an-operating-system)를 참조 하세요.


## <a name="deploy-to-pilot-devices"></a>파일럿 장치에 배포

Configuration Manager는 데스크톱 분석의 데이터를 사용 하 여 파일럿 및 프로덕션 배포를 위한 컬렉션을 만듭니다. 각 배포 단계 후 장치가 정상 상태 인지 확인 하려면 다음 절차를 사용 하 여 데스크톱 분석 통합 단계별 배포를 만듭니다.

1. Configuration Manager 콘솔에서 **소프트웨어 라이브러리**로 이동 하 여 **데스크톱 분석 서비스**를 확장 하 고 **배포 계획** 노드를 선택 합니다.  

2. 배포 계획을 선택 하 고 리본에서 **배포 계획 세부 정보** 를 선택 합니다.  

3. 리본에서 **단계별 배포 만들기** 를 선택 합니다. 이 작업은 단계별 배포 만들기 마법사를 시작 합니다.

    > [!Tip]  
    > 파일럿 컬렉션에 대해서만 클래식 작업 순서 배포를 만들려는 경우 **파일럿 상태** 타일에서 **배포** 를 선택 합니다. 이 작업을 수행 하면 소프트웨어 배포 마법사가 시작 됩니다. 자세한 내용은 [Deploy a task sequence](/sccm/osd/deploy-use/deploy-a-task-sequence)항목을 참조하세요.  

4. 배포의 이름을 입력 하 고 사용할 작업 순서를 선택 합니다. **기본 두 단계 배포를 자동으로 만드는**옵션을 사용 하 고 다음 컬렉션을 구성 합니다.  

    - **첫 번째 컬렉션**:이 배포 계획에 대 한 **파일럿** 컬렉션을 찾아 선택 합니다. 이 컬렉션에 대 한 표준 명명 규칙은 `<deployment plan name> (Pilot)`입니다.

    - **두 번째 컬렉션**:이 배포 계획에 대 한 **프로덕션** 컬렉션을 찾아 선택 합니다. 이 컬렉션에 대 한 표준 명명 규칙은 `<deployment plan name> (Production)`입니다.

    > [!Note]  
    > 데스크톱 분석 통합을 사용 하면 Configuration Manager는 배포 계획에 대 한 파일럿 및 프로덕션 컬렉션을 자동으로 만듭니다. 이러한 컬렉션을 사용 하려면 이러한 컬렉션을 동기화 하는 데 시간이 오래 걸릴 수 있습니다. 자세한 내용은 [문제 해결-데이터 대기 시간](/sccm/desktop-analytics/troubleshooting#data-latency)을 참조 하세요.<!-- 4984639 -->
    >
    > 이러한 컬렉션은 데스크톱 분석 배포 계획 장치용으로 예약 되어 있습니다. 이러한 컬렉션에 대 한 수동 변경은 지원 되지 않습니다.<!-- 3866460, SCCMDocs-pr 3544 -->  

5. 단계별 배포를 구성 하는 마법사를 완료 합니다. 자세한 내용은 [단계적 배포 만들기](/sccm/osd/deploy-use/create-phased-deployment-for-task-sequence)를 참조하세요.

    > [!Note]  
    > **지연 기간 (일) 후에이 단계를 자동으로 시작**하려면 기본 설정을 사용 합니다. 두 번째 단계를 시작 하려면 다음 조건을 충족 해야 합니다.
    >
    > 1. 첫 번째 단계는 성공에 대 한 **배포 성공 백분율** 조건에 도달 합니다. 이 설정은 단계적 배포에서 구성 합니다.
    > 1. 중요 하 고 중요 한 자산이 *준비*된 것으로 표시 하려면 데스크톱 분석에서 업그레이드 결정을 검토 하 고 확인 해야 합니다. 자세한 내용은 [업그레이드 결정이 필요한 자산 검토](/sccm/desktop-analytics/deploy-prod#bkmk_review)를 참조 하세요.
    > 1. 데스크톱 분석은 *준비* 조건을 충족 하는 모든 프로덕션 장치를 Configuration Manager 컬렉션으로 동기화 합니다.

> [!Important]  
> 이러한 컬렉션은 멤버 자격 변경 내용에 따라 계속 동기화 됩니다. 예를 들어 자산의 문제를 식별 하 고이를 알 **수**없는 것으로 표시 하면 해당 자산이 있는 장치가 *준비* 조건을 더 이상 충족 하지 않습니다. 이러한 장치는 프로덕션 배포 컬렉션에서 삭제 됩니다.


## <a name="monitor"></a>모니터

### <a name="configuration-manager-console"></a>Configuration Manager 콘솔

배포 계획을 엽니다. **업그레이드 결정 준비-전체 상태** 타일은 배포 계획의 상태에 대 한 요약을 제공 합니다. 이 상태는 파일럿 및 프로덕션 컬렉션 둘 다에 대 한 상태입니다. 장치는 다음 범주 중 하나에 포함 될 수 있습니다.

- **최신: 장치가**이 배포 계획에 대 한 대상 Windows 버전으로 업그레이드 됨

- **업그레이드 결정 완료**: 다음 상태 중 하나입니다.
    - **수정** 이 **준비** 되었거나 준비가 된 중요 한 자산이 있는 장치
    - 장치 상태가 **차단**됨, [**장치 교체**](/sccm/desktop-analytics/about-deployment-plans#plan-assets) 또는 **장치 다시 설치**

- 검토 **되지 않음**: 중요 한 자산이 **검토 되거나 검토 되지 않은** 장치가 **진행 중입니다** .

다음 작업을 통해 **파일럿 상태** 및 **프로덕션 상태** 타일에서 장치 상태가 업데이트 됩니다.

- 호환성 평가에 대 한 변경을 수행 합니다.
- 장치가 대상 버전의 Windows로 업그레이드 됨
- 배포 진행

다른 작업 순서 배포와 동일한 배포를 모니터링 하는 Configuration Manager 사용할 수도 있습니다. 자세한 내용은 [OS 배포 모니터링](/sccm/osd/deploy-use/monitor-operating-system-deployments)을 참조하세요.


### <a name="desktop-analytics-portal"></a>데스크톱 분석 포털

[데스크톱 분석 포털](https://aka.ms/desktopanalytics) 을 사용 하 여 배포 계획의 상태를 볼 수 있습니다. 배포 계획을 선택한 다음 **계획 개요**를 선택 합니다.

![데스크톱 분석의 배포 계획 개요 스크린샷](media/deployment-plan-overview.png)

**파일럿** 타일을 선택 합니다. 파일럿 배포의 현재 상태를 요약 합니다. 이 타일에는 시작 되지 않음, 진행 중, 완료 됨 또는 문제 반환 장치 수에 대 한 데이터도 표시 됩니다.

오류 또는 기타 문제를 보고 하는 모든 장치는 오른쪽의 파일럿 세부 정보 영역에도 나열 됩니다. 보고 된 문제에 대 한 세부 정보를 보려면 **검토**를 선택 합니다. 이 작업은 보기를 **배포 상태** 페이지로 변경 합니다.

**배포 상태** 페이지에는 다음 범주의 장치가 나열 됩니다.

- 시작 안 됨
- 진행 중
- 완료
- 주의가 필요한 장치
- 주의가 필요한 문제

**주의 필요** 범주는 동일한 정보를 표시 하지만 다르게 정렬 됩니다.

검색 된 문제에 대 한 자세한 정보를 보려면 각 보기에서 특정 목록을 선택 합니다.

이러한 배포 문제를 해결 하는 동안 대시보드는 계속 해 서 장치의 진행 상황을 표시 합니다. 장치 이동이 **완료**될 때까지 **주의가 필요** 하다 면 업데이트 됩니다.
